## 3. 验收脚本（手动 E2E）

### 3.1 脚本 A：主路径鼠标移动流畅度

1. 打开应用后依次进入 `/lens`、`/timeline`、`/intents`、`/settings`。
2. 每页持续移动鼠标 30 秒（覆盖列表项、按钮、边缘区域）。
3. 记录是否出现持续卡顿、明显掉帧或指针反馈迟滞。

### 3.2 脚本 B：点击反馈时延

1. 在上述页面各执行 20 次常见点击（按钮/切换/弹层开关）。
2. 观察点击后可见反馈（状态变化、弹层显示）是否出现明显延迟峰值。
3. 记录“无感/轻微/明显”三级结论与备注。

### 3.3 脚本 C：高频列表操作

1. 在时间线与会话列表执行快速上下滚动 + hover + 选中切换。
2. 连续切换 50 次条目，观察交互稳定性与视觉连贯性。

## 4. 验收标准（签收口径）

- Win10 与 Win11 在脚本 A/B/C 下均明显优于优化前记录。
- 无功能回归：采集、会话、截图、日报告、设置操作均保持可用。
- macOS 主路径回归通过。
- 文档归档完整（环境、脚本、结果、结论、已知限制）。

## 5. 风险与降级

| 风险 | 降级策略 |
|------|----------|
| 视觉弱化影响观感 | 仅 Windows 启用性能分支，保留 macOS 现有视觉 |
| 单机驱动差异导致结论波动 | 按机器分层记录，不做跨机绝对对比 |
| 局部优化引发样式偏差 | 优先小步提交，按页面回归逐项放量 |
