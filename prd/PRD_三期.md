# 🦞 TimeLens - 三期产品需求文档 (V2.0)

## 1. 产品定位与演进目标
在一期（数据采集闭环）和二期（多模态 AI 深度赋能）顺利落地的基础上，TimeLens 已经成为一款强大的单机个人时间追踪与分析工具。
三期的核心战略是**“破界与连接”**，打破单台设备的孤岛，并将视阈从“个人”拓展至“团队”，从“时间消耗”延伸至“实质产出”。

**核心目标：**
1. **多设备无缝漫游（Cross-device Sync）**：满足现代知识工作者（如公司用 Mac，家里用 Windows）的多端协同需求。
2. **生产力生态互联（Ecosystem Integrations）**：将「时间输入」与「工作输出」挂钩，衡量真实的生产力 ROI。
3. **团队效能洞察（Team Analytics B2B探索）**：在绝对保障员工隐私的前提下，为研发/设计小团队提供效能瓶颈分析，寻找商业化突破口。

## 2. 核心用户场景 (User Stories)

### 场景一：多端办公的无缝衔接
> **用户**：全栈工程师 小李
> **痛点**：白天在公司用 Mac 写后端，晚上回家用 Windows 电脑处理前端需求，当前版本只能分别看两份独立的报告，无法得知自己全天的整体工时。
> **三期解法**：通过云端时序数据库同步，小李的飞书每天只会收到一份合并后的完整日历，无论使用哪台电脑，时间线完美拼接，不重叠不遗漏。

### 场景二：开发产出的交叉验证
> **用户**：独立开发者 / 自由职业者 王工
> **痛点**：每天虽然显示 Coding 6小时，但具体是在改 Bug 还是开发新 Feature？效率如何？
> **三期解法**：TimeLens 关联了 GitHub 与 Notion。AI 在晚间报告中指出：“你今天下午 14:00-16:00 的深度编程时间，成功产出了 3 个 Commits，并关闭了 Notion 中的 2 个 P0 级 Bug，产能极高。”

### 场景三：团队敏捷教练的效能复盘
> **用户**：创业团队 Tech Lead 张总
> **痛点**：感觉团队最近推进很慢，但不知道是需求太难，还是沟通成本太高。
> **三期解法**：张总查看 TimeLens 团队看板（已开启隐私脱敏）。看板显示：“本周团队平均每天被会议打断 4 次，连续无干扰编程时间（Deep Work）平均不足 45 分钟。” 张总据此决定推行“无会议日”。

## 3. 详细功能模块说明

### 3.1 多设备云端漫游 (Multi-device Cloud Sync)
| 功能名称 | 描述说明 | 优先级 |
| :--- | :--- | :---: |
| **设备管理中心** | 允许用户在设置中查看、重命名、解绑已关联的设备（如 Mac-Office, Win-Home）。 | P0 |
| **时序数据融合** | 将多台设备上传的数据按时间戳进行云端 Merge。若多台设备同时活跃（例如开着 Mac 打电话，用 Win 查资料），AI 能够智能折叠或识别主导行为，避免时间重复计算。 | P0 |
| **端到端加密(E2EE)** | 引入 E2EE 机制或“自托管(BYOK)”选项（如支持 WebDAV、AWS S3），彻底消除用户对核心截图数据上云的隐私担忧。 | P1 |

### 3.2 产出生态深度互联 (Productivity Ecosystem Interconnect)
| 功能名称 | 描述说明 | 优先级 |
| :--- | :--- | :---: |
| **GitHub / GitLab 接入** | 授权获取用户的 Commits、PR、Issue 动态，时间戳与本地 Coding 行为对齐。 | P0 |
| **任务看板接入** | 支持 Notion、Jira、Linear 授权。将软件内的工作票（Ticket）状态流转，与对应时间段内的活跃应用关联。 | P1 |
| **日历同步双向对比** | 接入 Google Calendar / 飞书日历。对比“计划会议时间”与“实际会议耗时”，识别出“超时会议”或“临时拉起的未登记沟通”。 | P1 |
| **ROI / 吞吐量报告** | 在每日/每周飞书报告中新增模块：【今日产出】（如完成了 2 个 Issue，合并了 1 个 PR），结合 AI 给出耗时评估建议。 | P2 |

### 3.3 团队效能看板 (Team Effectiveness Dashboard - B端专属)
| 功能名称 | 描述说明 | 优先级 |
| :--- | :--- | :---: |
| **组织架构创建** | 允许用户创建 Team，邀请成员加入。成员需明确授权同意加入“团队视图”。 | P0 |
| **严苛的隐私脱敏机制** | **核心红线**：团队管理者**绝对无法**看到成员的具体屏幕截图、具体聊天窗口标题。所有上报到团队看板的数据，仅为“高维分类数据”（如：编程 4h，会议 2h，设计 1h）。 | P0 |
| **打断与专注度分析** | 统计团队的“平均碎片化时间”、“最长连续专注时长（Deep Work Time）”，帮助管理者优化排期。 | P1 |
| **会议成本计算器** | 自动折算团队在跨部门沟通上的时间消耗，给出“周度会议健康度诊断”。 | P1 |
| **团队群报告机器人** | 每周五下午自动向飞书群推送团队整体的时间效能周报，不针对个人，只针对群体大盘。 | P2 |

### 3.4 账户与订阅系统 (Account & Billing)
| 功能名称 | 描述说明 | 优先级 |
| :--- | :--- | :---: |
| **付费网关集成** | 接入 Stripe 等支付渠道，实现订阅制收费。 | P0 |
| **额度控制逻辑** | 限制免费用户的云端同步设备数（如 1 台设备），或限制云端数据保留时间（如 7 天），高级版无限制。 | P0 |

## 4. UI 界面与交互演进

### 4.1 客户端界面变化
- **侧边栏拓展**：新增 `My Devices`（我的设备）、`Integrations`（账号授权与集成）模块。
- **日历看板升级**：时间轴上不仅显示本地 App 使用记录，还会穿插显示外部事件。例如在 15:30 处插入一个 GitHub Icon：`✨ Pushed commit "fix auth bug"`，并由连线指向前两小时的 VSCode 时间块。

### 4.2 Web 端 / 团队看板 (新增)
- 面向团队管理者提供 Web 端 Dashboard，不再仅局限于本地 Tauri 客户端。
- 提供 `Team Overview`：柱状图堆叠显示团队每日精力分配（Coding vs Meeting vs Operations）。
- 提供 `Deep Work Score` 趋势线。

## 5. 技术架构演进需求

为了支撑三期，原有的轻量级后端需要进行重构和增强：

1. **分布式数据一致性（CRDTs 或 时间戳合并）**
   - 解决用户离线使用 A 设备，又使用 B 设备，恢复网络后数据上传的合并冲突。
2. **云端时序数据库升级**
   - 从目前的 PostgreSQL JSONB 存储，逐步向更专业的时序数据库（如 ClickHouse 或 TimescaleDB）迁移，支撑未来 B端海量的团队时序事件查询与聚合。
3. **OAuth 2.0 平台与外部 Webhook**
   - 后端需要实现完善的 OAuth 2.0 Client，用于接管 GitHub、Notion 等三方服务的 Token 刷新和生命周期管理。
   - 建立 Webhook 接收层，实时监听 GitHub 的 push event。
4. **团队多租户架构 (Multi-tenant Architecture)**
   - 数据库表结构需引入 `tenant_id` (组织/团队ID) 和 RBAC (Role-Based Access Control) 权限模型。

## 6. 商业模式设计与定价策略 (商业化试水)

三期是 TimeLens 开启商业化的关键阶段。采取 **PLG (Product-led Growth)** 模式，由好用的单机工具吸引极客用户，再由极客用户带入其所在的小团队。

| 版本 | 价格策略 | 核心权益限制 |
| :--- | :--- | :--- |
| **Free (个人免费版)** | $0 / 永久免费 | 单设备，纯本地存储（或云端同步限 7 天），每日基础 AI 飞书推送。 |
| **Pro (个人专业版)** | $5.99 / 月 | 多设备云端无缝漫游，30天以上云端数据备份，打通 GitHub/Notion，高级多模态 AI 深度洞察。 |
| **Team (团队企业版)** | $9.99 / 月 / 人 | 包含 Pro 所有功能 + 团队效能看板 + 组织级脱敏报告汇总 + 专用客户支持。 |

## 7. 里程碑与排期规划 (Roadmap)

- **M1 (第 1-2 周)：数据漫游与多端重构**
  - 完成设备注册、绑定逻辑。
  - 完成云端数据库合并多设备事件时间线的算法。
- **M2 (第 3-4 周)：打通产出生态**
  - 接入 GitHub & Notion OAuth。
  - AI prompt 升级，实现“耗时”与“产出节点”的匹配。
- **M3 (第 5-7 周)：团队版与隐私脱敏**
  - 搭建 Web 端团队管理控制台。
  - 实现严格的数据脱敏与 RBAC 权限隔离。
  - 开发团队级别的周报聚合逻辑。
- **M4 (第 8 周)：商业化与订阅网关**
  - 接入 Stripe，实现免费/专业/团队三个 Tier 的权限墙控制。
  - 内测与公测。
