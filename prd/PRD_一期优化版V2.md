# 🦞 TimeLens 一期产品设计文档（优化版）
## 文档说明
本文档基于TimeLens一期原始产品设计文档，针对页面设计落地中的逻辑不通、模块缺失、交互不符等问题，补充**设计落地细则、模块优化要求、交互逻辑规范**，核心锚定一期“建立高还原度隐私时序视觉数据湖、跑通数据获取-本地落盘-时序可视化闭环”的定位，不新增AI自动化分析等超范围功能，兼顾普通用户的使用体验与开发者的校准需求。

## 1. 核心定位 (Core Value)
一期（Phase 1）的产品定位是**“建立最丰满、高还原度且极度重视隐私的时序视觉数据湖”**。
在一期阶段，**暂不引入** AI 自动化分析功能。本阶段的终极使命是跑通从操作系统底层获取精细化活动数据，到本地安全落盘，再到以“图文并茂”的时间轴向用户（或开发者）进行汇报的完整闭环。
**落地核心要求**：所有设计优化均围绕“数据精准抓取、本地安全存储、多维度可视化、开发者可校准”展开，保证双模式体验分层、功能模块完整、交互逻辑顺畅。

## 2. 核心收集策略（重拾深层上下文）
### 2.1 深度全息文本记录
不要只停留于监听 `App Name`（如仅记录“使用 Chrome 2 小时”）。必须向下探索甚至精准抓取**深层上下文**，且落地时保证数据在时序面板中完整展示：
* **字段包含**：应用程序名、**具体的窗口完整标题**（例：`ChatGPT 聊天 - Google Chrome` 或 `bug修复方案.docx - Word`）、活动绝对时间戳、持续时长。
* **落地要求**：所有字段需在时序流列表中完整呈现，无信息截断，持续时长以`持续 XX'XX"`格式展示，窗口标题保留原生完整名称。

### 2.2 智能截图与抽帧 (Smart Screenshots)
坚持采用**智能截屏（抽帧）**而非全天候录视频，以兼顾上下文还原度与 CPU/GPU 性能开销，为后续大语言模型打基建，落地时需展示截图核心属性，支持开发者校准：
* **触发机制**：
  * **事件驱动**：当操作焦点（活动窗口）发生切换，或同一窗口内标题发生实质性变化时，立刻截取。
  * **轮询驱动**：当用户在当前窗口持续操作时，按预设时间阈值（如隔 10~20 秒 1 帧）截断。
* **落地要求**：
  1. 截图预览区需标注**触发类型**（事件驱动/轮询驱动）、**压缩格式**、**文件大小**、**提取策略明细**（如分辨率、触发条件）；
  2. 缩略图网格添加触发类型标识小图标（事件驱动标△、轮询驱动标○），支持快速识别；
  3. 截图支持放大/缩小/本地保存，方便目视核对。
  4. 能展示出最新一次截图记录时间，以及预计下次截图记录时间，能banner告知用户截图的原理和逻辑。

## 3. 双轨运行模式机制 (Dual-Mode System)
一期产品在启动与运行阶段必须内置**清晰的模式切换入口、明确的视觉风格区分、完整的模块布局**，便于开发者在早期校准和普通用户的监控验证，两种模式需实现**一键切换、状态记忆、模块自适应**。

### 3.1 正常模式 (Normal Mode)
* **核心特性**：极致克制的后台常驻模式，占用极低系统资源，突出“无感记录、本地加密、隐私安全”，无琐碎数据罗列打扰。
* **视觉设计**：
  1. 主界面核心区域展示**代表安全感的加密盾牌矢量插图**，插图下方标注核心安全标语：`100%本地存储 · 无任何外部网络请求`；
  2. 页面风格为**轻量化简约风**，以浅色系为主，减少视觉元素，仅保留核心功能入口。
* **功能模块**：
  1. 全局**模式切换入口**（右上角/顶部导航栏），标注“切换至开发者视界”，带盾牌状态标识；
  2. **时序总览快捷入口**：点击可快速查看极简版时序时长汇总，无细颗粒度数据；
  3. **设置入口**：包含本地存储路径查看、资源占用概况、基础配置（如截图时间阈值）；
  4. **安全提示浮层**：首次启动/模式切换时，弹出轻量提示框，说明“所有数据本地加密落盘，一期禁止任何外部API请求”。
* **资源要求**：后台运行时CPU占用率≤5%，内存占用量≤100MB，设置页面展示**平均资源占用概况**。

### 3.2 开发者视界 (Debug Mode / Developer View)
* **核心特性**：专为跑通一期抓取健壮性而打造的“开发者抓取看板层”，主打**左右分屏的玻璃拟态（Glassmorphism）暗色科技风**架构，所有校准、调试模块完整展示。
* **视觉设计**：
  1. 整体为**玻璃拟态暗色科技风**，满足“半透明+模糊背景+轻微边框”特征，文字/图标保证高对比度，避免视觉模糊；
  2. 模块间做清晰的视觉分隔，时序面板与图库回放区无重叠，支持窗口响应式适配；
  3. 所有操作按钮添加**hover/点击视觉反馈**，核心按钮（如直通黑盒）做醒目样式设计。
  顶部 集成**应用、意图/功能、跨度区段**三维筛选器，筛选结果实时刷新；
* **左侧·实时时序面板**：
  2. 流式展现**带时间戳、应用原生图标、耗时计算、窗口完整标题、功能Tag**的流水账，列表项支持点击高亮；
  3. 右侧**快捷键操作**，如筛选确认、列表翻页等，提升操作效率。
* **底部·走马灯图库回放**：
  1. 顶部：**大尺寸最新快照预览区**，添加**监控红框视觉效果**，附带快照大小、压缩格式、触发机制、提取策略明细；
  2. 底部：**按时间排序的缩略图网格（Gallery）**，按比例压缩展示同时间段截图，点击缩略图可切换主预览区内容，实现走马灯式回放；
  3. 主预览区支持**高清截图存根渲染**，无延迟加载，与左侧时序列表实现热更新联动。
* **直通黑盒交互**：
  1. 页面底部**永久固定悬浮**“100% 本地沙盒”的安全确认横幅（醒目色系字体）；
  2. 横幅右侧放置`[直通黑盒底层]`操作按钮，点击后**一键呼出访达/资源管理器**，并自动定位至真实 SQLite 及截图存放的文件夹；

### 3.3 模式切换规范
1. **入口**：全局唯一切换入口，位于页面右上角，标注当前模式名称，带对应视觉标识（正常模式-盾牌、开发者视界-科技风图标）；
2. **交互**：点击后**一键切换**，页面瞬间完成视觉风格、模块布局的自适应，无需重新加载；
3. **记忆**：系统自动记忆上一次使用的模式，下次启动时默认进入该模式；
4. **提示**：模式切换后，弹出轻量提示框（3秒自动消失），告知当前模式核心功能。

## 4. 细颗粒度的时序仪表面板 (Timeline Dashboard)
突破传统产品只汇总圆饼图的时长罗列，打造**深度可视化、带有行为树层级、多维度筛选、快照联动**的图文详情面板，同时保证操作便捷性、数据可读性，该面板为**开发者视界核心模块**，正常模式仅展示极简汇总版。

### 4.1 多维度的筛选控制栏 (Multi-dimensional Filters)
面板顶部固定布局，按**应用维度→意图/功能维度→跨度区段维度**的逻辑排序，支持**多条件组合筛选**，筛选结果实时渲染，无延迟等待。
* **1. 应用维度 (By App)**：
  1. 提供**高频应用勾选式标签**，标签带应用原生图标，支持一键全选/取消；
  2. 添加**应用搜索框**，支持模糊搜索小众应用，快速定位；
  3. 标签按使用频率从高到低排序，支持自定义标签排序。
* **2. 意图/功能维度 (By Intent/Type)**：
  1. 系统底层通过**简易查表（字典映射应用名至分类）**，提炼固定筛选条块：`全部分类`, `编码开发`, `研究检索`, `AI 辅助`, `文档撰写`, `通讯沟通`；
  2. 筛选条块为**一键切换式**，点击即可过滤对应分类数据，支持多分类组合筛选；
  3. 提供**自定义分类入口**，开发者可新增/编辑/删除分类，修改应用与分类的映射关系。
* **3. 跨度区段维度 (By Time Block)**：
  1. 包含**常规日历选择器**，支持按日/周/月筛选；
  2. 新增**高度符合人类作息的切块筛选按键**：`全天`, `上午 (06-12)`, `中午 (12-14)`, `下午 (14-18)`, `晚上 (18-24)`；
  3. 作息切块按键为一键切换式，与日历选择器联动，支持组合筛选（如“2026-03-19 + 上午”）。

### 4.2 直观易读的平铺时序流 (Flat Timeline List)
筛选控制栏下方核心区域，流式展示时序数据，列表项具备**高度视觉重心**，信息完整、层级清晰，支持鼠标悬停详情展示、点击快照联动。
* **列表项必含元素**（从左到右排序）：
  1. 时间信息：起止时间（如10:05-10:17）+ 活动绝对时间戳（鼠标悬停展示）；
  2. 时长信息：持续时长 (例: `持续 12'04"` )，醒目字体展示；
  3. 应用标识：应用原生矢量图表徽章，与应用名联动；
  4. 核心内容：应用程序名 + 具体的窗口完整标题，无信息截断；
  5. 功能标签：末尾跟随的功能性 Tag (如 `AI辅助`)，与意图/功能维度分类一致，不同分类用不同颜色区分。
* **视觉与交互要求**：
  1. 列表项做**隔行变色**处理，提升可读性；
  2. 鼠标悬停列表项，展示**悬浮详情框**，包含该条数据的所有核心信息（如截图触发类型、存储路径）；
  3. 点击列表项，实现**高亮选中**，同时触发右侧主快照区热更新联动。

### 4.3 视域快照热更新联动
实现**左侧时序列表与右侧主快照区的无延迟联动**，核心满足“点击即渲染、精准匹配时间戳”，支持目视交叉比对数据与截图的一致性。
* **联动逻辑**：
  1. 点击左侧任意一条时间流水账，右侧主快照区**立刻热响应**，渲染该条数据对应时间戳下截取的高清图片存根；
  2. 若该时间戳无截图，主快照区展示**无截图提示**，并标注原因（如未触发截图条件、截图失败）；
  3. 切换列表项选中状态，主快照区同步切换对应截图，无缓存残留。
* **展示要求**：
  1. 主快照区渲染的截图为**原始压缩后的高清版本**，支持放大/缩小/拖拽查看细节；
  2. 截图下方同步展示**该条数据的核心信息**（与列表项一致），实现数据与截图的双重核对。

## 5. 绝对本地的安全地基 (Local Storage Only)
将“100%本地沙盒、绝对隐私、无外部上传”作为一期核心卖点，通过**视觉标识强化、操作入口落地、存储信息透明**，让普通用户感知安全，让开发者可验证、可校准。

### 5.1 100% 本地沙盒（落地细则）
* **存储路径**：配置极度明确的本地存储路径，落地时需在产品中**完整展示、支持一键复制**：
  1. 截图存储：`~/.timelens/data/shots/`
  2. 时序数据库：`~/.timelens/data/db.sqlite`
* **网络限制**：**一期禁止任何外部 API 网络请求**，阻断任何上传未授权服务器的操作，产品中添加**网络检测机制**，若检测到外部网络请求，立即终止并弹出警示。
* **本地加密**：所有数据（含截图、时序数据库）均实现**本地加密落盘**，加密方式在设置页面做简洁说明，不展示复杂技术细节，避免用户困惑。

### 5.2 安全视觉与操作落地
* **全模式安全标识**：
  1. 正常模式：主界面核心位置展示加密盾牌插图，搭配安全标语，贯穿整个使用流程；
  2. 开发者视界：底部永久悬浮“100%本地沙盒”安全横幅，与直通黑盒按钮联动，强化安全认知。
* **存储信息透明化**：
  1. 设置页面添加**“本地存储信息”专属模块**，展示截图、数据库的具体存储路径，支持**一键复制路径**；
  2. 展示**本地存储占用量**，按截图/数据库分类统计，支持一键清理历史数据（需二次确认）。
* **安全验证入口**：
  1. 开发者视界中`[直通黑盒底层]`按钮，一键定位本地存储文件夹，让开发者直观验证数据本地落盘；
  2. 设置页面添加**“安全验证”入口**，点击后可检测本地存储的加密状态、网络请求限制状态，生成简单的验证报告。

## 6. 通用设计与交互规范
### 6.1 响应式适配
所有页面/模块均支持**窗口缩放自适应**，适配不同屏幕尺寸（如笔记本13寸、台式机27寸），避免小屏幕下模块重叠、信息截断，大屏幕下留白过多。

### 6.2 快捷键支持
为开发者视界的核心操作添加**快捷键**，提升操作效率，所有快捷键在产品中提供**快捷键说明入口**（如设置页面/开发者视界帮助浮层），支持自定义快捷键。

### 6.3 操作反馈
所有核心操作（如模式切换、筛选、直通黑盒、截图放大）均添加**清晰的视觉/交互反馈**（如按钮变色、加载动画、提示框），避免用户无感知、误操作。

### 6.4 容错机制
1. 截图失败/数据抓取异常时，在对应模块展示**明确的异常提示**，并标注可能的原因（如系统权限不足、应用兼容问题）；
2. 点击无效操作（如无截图时点击放大），展示轻量提示，无错误弹窗，保证使用体验。
