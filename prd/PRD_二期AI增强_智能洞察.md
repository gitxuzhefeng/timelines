# TimeLens 二期产品设计文档 — AI 增强（独立版本）

> **版本**：v1.0  
> **日期**：2026-04-01  
> **定位**：在 **二期基础（v1）** 已交付的前提下，增加 **opt-in 的 LLM 叙事层**：BYOK、`generate_daily_report(with_ai: true)` → `ai_enhanced` 报告、隐私弹窗、失败降级与双版本并存 UI。  
> **前置依赖**：`PRD_二期基础_智能洞察.md` v1.0 所定义的事实层、表结构、`daily_analysis` 语义 **全部可用**；本册 **不改变** 指标计算公式（见 `二期_分析指标字典.md` 首行说明）。  
> **技术方案**：`rule/TimeLens_TechArch_Phase2_AI增强.md`  
> **配套文档**：`二期AI增强_里程碑与验收计划.md`、`二期AI增强_产品研发任务计划表.md`

---

## 一、目标与原则

| 原则 | 说明 |
| --- | --- |
| **增强不替代** | AI 只消费 `daily_analysis`（及允许的聚合视图），**不可**改写数字；事实层报告始终可独立生成 |
| **默认关闭** | 总开关默认关；**首次开启**须隐私说明与用户确认 |
| **BYOK** | 用户自配 API Key、Base URL、模型名；应用不内置 Key |
| **可下钻** | 叙事中的数字与事实层锚点一致（产品可定义 `[§章节]` 等） |

---

## 二、隐私与出境红线

与合并版 `PRD_二期_智能洞察.md` **§4.1 输入约束** 一致：

| 允许出境 | 禁止出境 |
| --- | --- |
| `daily_analysis` 聚合 JSON | `raw_events` 全表、窗口标题原文、剪贴板正文、按键序列 |
| Intent 名、应用名（规范内） | 密码、URL token 等 |

**截图 / OCR**：与 **`PRD_二期截图增强_智能洞察.md` v3.0** 对齐——默认本地处理；向模型出境须 **单独 opt-in**；优先 **脱敏后的 OCR 结构化摘要**，非必要不传原图；**截图黑名单**与敏感过滤以该文档 §五、§七 为产品准绳（合并版 §4.3 / §6.2 可作历史对照）。

---

## 三、Prompt 策略

与合并版 **§4.1 AI Prompt 策略** 一致：固定 system prompt + 结构化 JSON；**禁止**开放式闲聊接口作为 P0。

---

## 四、报告与存储

- `daily_reports.report_type = 'ai_enhanced'`  
- 与 `fact_only` **并存**；UI **可切换**查看  
- `ai_model`、`ai_prompt_hash` 字段用于追溯

报告章节在事实段之后附加 **AI 解读**（建议 Markdown 引用块样式区分）。

---

## 五、失败与降级

- API 错误、超时、无效 Key：**明确错误提示** + **回退**到 `fact_only`（或保留上次成功 `ai_enhanced` 由产品定）  
- 网络不可达：同上

---

## 六、UI 增量（相对二期基础）

| 区域 | 增量 |
| --- | --- |
| **设置** | API Key（安全存储）、模型、Base URL；首次开启隐私弹窗 |
| **复盘** | 「事实 / AI 增强」切换（仅当存在 `ai_enhanced` 或开关开启时显示逻辑由产品细化） |
| **工程** | `reqwest` 或等价 HTTP **仅**在 AI 模块；`cargo` feature 隔离策略见技术方案 |

---

## 七、验收主轴（摘要）

详细 Given/When/Then 见 **`二期AI增强_里程碑与验收计划.md`**，对应原合并版 **M5-01～M5-07、M5-10** 及 **DG-08** 在「AI 开启」下的补充场景。

| 主题 | 要点 |
| --- | --- |
| 首次开启 | 隐私弹窗 → 再进入 Key 配置 |
| BYOK | 持久化；日志不脱敏 Key |
| 生成 | `ai_enhanced` 内容非空；章节结构符合 PRD |
| 数字一致 | AI 文中数字与 `daily_analysis` 一致 |
| 关闭 AI | 无外向请求；事实报告完整 |
| 双版本 | 同日可存 fact + ai_enhanced |
