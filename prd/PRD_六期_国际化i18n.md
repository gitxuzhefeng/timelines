# PRD：六期 — 国际化（i18n）支持（V6）

**版本**：v1.0 · 2026-04-19
**上游基准**：`prd/PRD_五期_Windows交互性能优化.md`
**关联实现**：桌面端 Tauri + React（`project/`）；README（仓库根目录）

---

## 1. 背景与问题

当前版本所有 UI 文案、页面标签、交互提示均硬编码为中文，README 也仅有中文版本。随着项目对外开放，英语用户无法正常使用，也无法通过 README 了解产品。

本迭代聚焦**中英双语国际化**，不引入第三语言，不改变任何业务逻辑与数据结构。

---

## 2. 本迭代使命

| 阶段 | 本迭代聚焦 |
|------|------------|
| 一至五期 | 数据底座、智能洞察、正式壳、体验优化、Windows 性能（已完成） |
| **六期** | **中英双语 i18n：UI 全页面文案可切换、README 双语、语言偏好持久化** |

---

## 3. 用户与场景

### 3.1 主路径场景

- 英语用户安装后，界面默认显示英文（跟随系统语言）。
- 中文用户安装后，界面默认显示中文，体验与现有版本一致。
- 用户可在设置页手动切换语言，切换后立即生效，重启后保持。
- GitHub 访客打开 README，可选择查看中文或英文版本介绍。

### 3.2 边界

- 本期仅支持 **zh-CN**（简体中文）和 **en**（英文）。
- 不做繁体中文、日语等其他语言。
- OCR 识别语言、AI 分析语言不在本期范围内（仅 UI 文案）。
- 数据库内容（应用名、窗口标题等）为系统原始数据，不做翻译。

## 4. 功能与能力清单

### 4.1 P0（本迭代必达）

| 能力 | 说明 |
|------|------|
| i18n 基础设施 | 引入 `react-i18next` + `i18next`，建立 `zh-CN` / `en` 两套 JSON 词条文件 |
| 全页面文案覆盖 | 11 个页面（TodayLens、Timeline、Sessions、DailyReport、OcrSearch、OcrEval、IntentManage、Health、Settings、Recap、SettingsShell）及 4 个组件的所有硬编码中文替换为 `t()` 调用 |
| 语言自动检测 | 首次启动时读取系统语言（`navigator.language`），匹配 zh/en，无匹配则默认 en |
| 语言手动切换 | 设置页新增语言选择项（中文 / English），切换立即生效 |
| 语言偏好持久化 | 用户选择存入 Tauri 本地设置（与现有 settings 机制对齐），重启后恢复 |
| README 双语 | 根目录 README.md 提供中英双语内容（同一文件，顶部语言切换锚点） |

### 4.2 P1（可并入本期）

| 能力 | 说明 |
|------|------|
| 托盘菜单国际化 | 托盘右键菜单文案（打开主窗口、开始采集、停止采集、退出）跟随语言设置 |
| 日期/时间格式本地化 | 根据语言切换日期显示格式（zh: `YYYY年MM月DD日`，en: `MMM DD, YYYY`） |

### 4.3 明确不做（本迭代）

| 项目 | 说明 |
|------|------|
| 第三语言 | 不引入 zh-TW、ja、ko 等，框架预留扩展点即可 |
| 后端/Rust 侧文案 | 日志、错误码等 Rust 侧字符串不做翻译 |
| OCR / AI 语言切换 | 识别与分析语言不在本期范围 |
| 数据内容翻译 | 应用名、窗口标题等采集到的原始数据不做处理 |
| RTL 布局 | 不支持阿拉伯语等从右到左语言 |

## 5. 关键指标与验收阈值

### 5.1 验收口径（P0）

1. **覆盖率**：11 个页面 + 4 个组件中，无肉眼可见的硬编码中文（英文模式下）。
2. **切换即时性**：设置页切换语言后，当前页面文案无需重启即刷新。
3. **持久化**：重启应用后，语言设置与上次一致。
4. **自动检测**：系统语言为英文的设备，首次启动默认显示英文。
5. **无回归**：中文模式下，所有页面文案与五期版本一致，功能无异常。
6. **README**：英文读者可通过 README 完整了解产品功能与安装方式。

### 5.2 记录模板（必填）

- 测试设备系统语言设置
- 切换前/后截图对比（至少覆盖 TodayLens、Settings、Sessions 三个页面）
- 重启后语言持久化验证记录

---

## 6. 方案摘要（实现约束）

1. **词条文件结构**：`project/src/i18n/locales/zh-CN.json` 与 `en.json`，按页面分组（`todayLens.*`、`timeline.*`、`settings.*` 等），便于维护与扩展。
2. **初始化**：在 `main.tsx` 或 `App.tsx` 顶层初始化 `i18next`，语言来源优先级：持久化设置 > 系统语言 > `en`。
3. **持久化对接**：复用现有 Tauri settings 命令（`get_settings` / `set_settings`），新增 `language` 字段，避免引入新的存储机制。
4. **托盘文案（P1）**：Rust 侧托盘菜单在语言切换时通过 Tauri 命令重建菜单项，或在前端切换时通知 Rust 侧更新。
5. **低侵入原则**：仅替换文案，不改变组件结构、样式与业务逻辑。

---

## 7. 验收要点（Given / When / Then 摘要）

1. **Given** 系统语言为英文，**When** 首次启动 TimeLens，**Then** 界面默认显示英文。
2. **Given** 中文模式，**When** 进入设置页切换为 English，**Then** 所有可见文案立即切换为英文，无需重启。
3. **Given** 切换为英文后关闭并重启应用，**When** 应用启动，**Then** 界面仍显示英文。
4. **Given** 英文模式，**When** 浏览 TodayLens、Sessions、DailyReport 等页面，**Then** 无中文硬编码文案残留。
5. **Given** 英文 README，**When** 英语用户阅读，**Then** 可完整理解产品定位、功能与安装方式。
6. **Given** 中文模式，**When** 执行全部主路径操作，**Then** 功能与五期版本无回归。
