## 1. 背景与问题（PRD_BACKGROUND）

经过 1-11 期，TimeLens 的功能能力已基本完整：采集 → 浏览 → 分析 → 导出 → AI 助手 → 智能提醒 → 专注模式。但在实际使用中，**信息呈现层面的体验问题**逐渐暴露：

- **主页（今日透视）信息密度低、不动**：「事实报告摘录」与「数据管线」占了首页大量空间，且每次打开内容差异不大，用户失去新鲜感与回访动机
- **打断来源不直观**：柱状图须悬停才能看到具体应用，移动端/快速浏览时信息不可见
- **时间线页面信息过载**：每次窗口切换都记录一行，一天可能上百行，长时间向下滚动疲劳
- **时间线时序方向反直觉**：用户想看"最近做了什么"时不希望从早上滑到现在
- **AI 助手未跟随界面语言**：UI 中文、AI 回答英文，体验割裂
- **设置页拥挤**：高级开发型选项（OCR 管线参数等）和常用项目混在一起
- **关于页有冗余**：「支持与赞助」与个人项目定位不符
- **开发工具入口含旧/弱功能**：影响用户对生产力工具的认知

## 3. 用户与场景（PRD_USERS）

### 3.1 核心用户画像

- **重度日活用户**：每天多次打开 TimeLens 看状态，希望首页有变化、有新洞察
- **快速浏览型用户**：滑动几下就要做出判断，不想悬停、不想放大鼠标精度
- **多语言用户**：界面中文，期望 AI 也用中文回答

### 3.2 主路径场景

**场景 A**：用户中午打开「今日透视」，第一眼就看到上午心流评分、专注次数、Top 1 应用，而不是看到一段固定的事实摘录。

**场景 B**：用户晚上想看"最近 1 小时干了啥"，打开时间线，最上面就是刚刚结束的会话，往下是更早的；连续在 IDE 写代码的 30 分钟会被合并为一组。

**场景 C**：用户在中文界面问助手「今天专注了多久」，助手用中文回答。

**场景 D**：用户进入设置页，看到的是清爽的核心选项（主题、语言、提醒、AI 等），而 OCR 管线参数等高级项隐藏在「高级设置」（须开启开发者模式）中。

## 6. 数据与交互定义

### 6.1 时间线分组规则（前端）

输入：`Session[]`（按 start_ms 升序）。

合并条件：相邻两个 session 同时满足
- 同 `app_name`
- 间隔 ≤ 3 分钟
- 累计组时长 < 90 分钟（避免一组太长无意义）

输出：`SessionGroup[]`：
```
{
  id: string,
  start_ms: number,
  end_ms: number,
  app_name: string,
  intent?: string,
  total_duration_ms: number,
  session_count: number,
  sessions: Session[],   // 原始细分
}
```

### 6.2 开发者模式

复用现有 `devModeStore`。在「高级设置」section 顶部展示开发者模式开关；关闭时下方内容不显示。

## 7. 验收要点（PRD_ACCEPTANCE）

1. **L1**：进入今日透视，「事实报告摘录」与「数据管线」不再占据主视觉中心；动态指标摘要清晰

3. **L3**：时间线第一条是当日最近的会话
4. **L4**：连续 IDE 会话在分组视图中合并为一个块，块标题包含累计时长与会话数
5. **L5**：右上角切换按钮可在「分组视图」和「完整视图」间切换
6. **L6**：中文界面下问助手 → 助手用中文回答；英文界面 → 英文回答
7. **L7**：设置页有「高级设置」section，关闭开发者模式时只显示开关；开启后展示 OCR 管线参数等高级项
8. **L8**：设置页顶部紧凑栏一行包含语言、主题、日期选择
9. **L9**：关于页无「支持与赞助」；反馈段落含邮箱 xuzhfeng222@gmail.com 且为可点击 mailto 链接
10. **L10**：左侧主菜单包含「OCR 检索」；开发工具不再显示「复盘」入口
11. **i18n**：所有新增/修改文案双语完整
12. **回归**：智能提醒、专注模式、AI 助手、日报、周报等已有功能正常

## 8. 已知限制

- 时间线分组规则为前端启发式，特殊场景（如长会议中频繁切换文档）可能合并到同一块，提供「完整视图」兜底
- 开发者模式入口位于设置页内，新用户可能不会主动开启（符合预期）
