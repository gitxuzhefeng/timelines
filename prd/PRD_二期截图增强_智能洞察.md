## 〇、背景与问题（简述）

窗口标题与 App 名只能回答「在哪个软件」；同 App 内多标签、多文档、多任务时，用户复盘时常 **想不起当时在做什么**。截图已存在，但未结构化，无法检索、无法稳定进入报告叙事。

**本迭代手段**：对已采集帧做 **本地 OCR**，得到 **可检索、可摘要、可脱敏** 的文本信号，与现有会话对齐——**不**把 OCR 包装成「全自动读心」或「替代人工填工时」。

---

## 一、OCR 在本产品中的「核心价值」定义（正式）

下列三项为 **OCR 能稳定创造、且与 TimeLens 定位一致** 的用户价值；**其余能力若非以下延伸，本迭代不承诺**。

### 1.1 核心能力 A：任务锚点（Session Anchor）

**定义**：从屏幕可见文字中提取 **短实体与标题片段**（如工单号、文档标题、浏览器页签可见标题、编辑器路径片段、仓库/issue 可见文案），与会话时段绑定，用于回答「这段时长 **更可能** 对应哪件具体事」。

**用户可感知产出**：会话详情中 **一行摘要**（示例：`JIRA PROJ-123 · 用户认证`），来源标注为「来自屏幕文字」；缺失则为空或 `unknown`，**不编造**。

**边界**：不保证覆盖所有界面（图片型 UI、游戏、全视频、非拉丁字符、极小字号等会缺失）；**不等于**自动项目管理或自动工时法律效力。

### 1.2 核心能力 B：可检索回忆（Keyword Recall）

**定义**：对 OCR 结果建 **本地全文/关键词索引**（如 SQLite FTS 或等价），用户按 **关键词** 检索，返回 **时间点 + 会话 + 可选缩略图**。

**用户可感知产出**：「我记得页面上出现过某错误码/某文件名」→ 能搜到 **何时、在哪个会话**。

**边界**：本迭代承诺 **关键词匹配**，**不承诺**「自然语言语义搜索」（依赖向量与模型，属后续能力）；检索范围受黑名单与敏感过滤约束。

### 1.3 核心能力 C：AI / 报告的最小必要上下文（Structured Gist）

**定义**：在用户 **单独授权** 且通过黑名单与脱敏后，向 `ai_enhanced` 或导出内容提供 **结构化摘要**（实体列表 + 极短标题片段 + 场景粗标签 + 置信度），**优先文本**、**非必要不传图**。

**用户可感知产出**：AI 叙事中可出现「结合当日屏幕可见标题，你在 … 相关界面上停留较久」类表述，且与事实时长 **分块排版**、标明推测。

**边界**：不替代 `daily_analysis` 数字；出境内容须符合 `PRD_二期AI增强_智能洞察.md` 红线。

## 二、本迭代剔除的「伪需求」与原因（范围减法）

下列项曾在设想或 v2 草案中出现，经 **需求强度 × OCR 可达成度 × 隐私/维护成本** 评审，**不作为本二级迭代承诺**（可记入远期 Backlog，不写入验收）。

| 剔除项 | 为何视为伪需求或延后 |
| --- | --- |
| **「自动工作流叙事」**（如需求→编码→测试→回 Jira 的链条） | 强依赖时序推理与 LLM，非 OCR 单独可交付；易过度解读、难验收 |
| **「同 App 内深度/浅度精准切分」**（如 Chrome 内学习 vs 摸鱼分钟级） | 仅靠 OCR + 简单规则 **误报率高**；用户信任成本大 |
| **「自动卡点/阻塞检测」**（如在某错误上卡了 40 分钟） | 错误形态跨应用差异极大，规则脆弱；易与正常调试混淆 |
| **「会议智能拆分」**（从 Zoom/Teams UI 读标题、场次数） | 会议软件 UI 变动频繁，OCR **脆弱**；通知+窗口标题已部分覆盖「有会议」事实 |
| **「自然语言 / 语义搜索屏幕历史」** | 需要本地嵌入或出境模型，超出本迭代「OCR + 索引」边界 |
| **「自动工时单 / 法律效力级工时证明」** | 用户期望与系统能力_gap 大；合规场景应明确 **人工确认** |
| **「学习轨迹 / 技能图谱 / 个人知识库」** | 属于长期产品愿景，非截图二级迭代必达 |
| **纯视觉布局推断**（无文字时的深潜判定） | 非 OCR 核心；准确率低时不纳入本迭代口径 |

## 三、用户场景（收敛后）

仅保留与 **§一 三项核心能力** 直接相关的场景。

| ID | 场景 | 说明 |
| --- | --- | --- |
| **C1** | 日终/周后「这段到底在干嘛」 | 会话一行摘要 + 可点入 OCR 关键词片段（脱敏展示） |
| **C2** | 同 App 多任务 | 靠可见标题/票号等 **锚点** 区分，**不承诺** 分钟级子任务切割 |
| **C3** | 「我记得屏幕上出现过某字符串」 | **关键词检索** 定位时间与会话 |
| **C4** | 开启 AI 后报告更可读 | 在授权下注入 **结构化 Gist**，不改数字 |
| **C5** | 敏感内容绝不外流 | 黑名单 + 敏感模式过滤 + 默认本地 + 出境单独勾选 |

## 四、需求分层（精简）

### 4.1 显性需求（本迭代承接）

| ID | 需求 | 对应核心能力 |
| --- | --- | --- |
| D1 | 复盘时看到 **比 App 名更具体的一行说明** | A |
| D2 | 能 **搜索** 屏幕上出现过的 **关键词** | B |
| D3 | 敏感 App/界面 **不进 OCR、不出境** | 安全基线 |
| D4 | 不占盘、不卡（异步、抽样、可清理） | 工程约束 |
| D5 | 开 AI 时 **少传图、多传脱敏文字摘要** | C |

### 4.2 真实需求（JTBD，合并为两条）

1. **回忆**：快速想起「那段时间在做什么具体事」，而不只靠记忆与 App 名。  
2. **检索**：凭零碎线索（文件名、错误码、票号片段）找回 **时间点与会话**。

## 五、产品功能清单（正式分级）

### 5.1 原则（不变更二期 AI 红线）

| 原则 | 说明 |
| --- | --- |
| 增强不替代 | 不修改 `daily_analysis` 公式；OCR 为可选层 |
| 文本优于图片 | 出境与叙事优先 **结构化 OCR 摘要** |
| 默认本地 | OCR 与索引默认本机；出境须单独 opt-in |
| 宁缺勿乱 | 无把握则 `unknown` / 不展示，禁止「确定语气」编造 |

### 5.2 P0 — 本二级迭代 **必达**

| ID | 功能 | 验收要点 |
| --- | --- | --- |
| **S0-1** | **OCR 总开关 + 首次说明** | 默认关；开启前独立说明；关闭后停止新 OCR |
| **S0-2** | **黑名单与敏感过滤** | 命中帧不 OCR 或整帧 redacted；不出境 |
| **S0-3** | **本地 OCR 管线** | 异步处理已采集帧；结果写入本地存储（如 `session_ocr_context` / 等价） |
| **S0-4** | **会话一行摘要（任务锚点）** | 有关键实体/标题则展示一行；无则空或 unknown；可关 |
| **S0-5** | **与 AI 增强衔接** | BYOK 且用户勾选「OCR 摘要可出境」时，仅传 **脱敏结构化摘要**；默认不传图 |

### 5.3 P1 — 本二级迭代 **可选交付**（排期允许时）

| ID | 功能 | 说明 |
| --- | --- | --- |
| **S1-1** | **本地关键词检索** | 对 OCR 文本 FTS；结果 → 时间 + 会话 + 可选缩略图 |
| **S1-2** | **实体归集视图（辅助，非工时证明）** | 用户可配置 **正则/前缀/别名**（如票号模式），按日汇总「命中会话时长」；导出为 **草稿**，须提示人工核对 |

### 5.4 本迭代 **不做**（与 §二 一致）

实时录屏直播、远程查看、内置多模态 Key、语义搜索、自动工作流故事、自动深潜切分、自动卡点检测、会议 UI 智能拆分、技能图谱、法律效力级工时认证。

## 七、边界与约束（摘要）

- **隐私**：黑名单第一道；OCR 结果第二道敏感模式扫描；日志禁止完整屏幕文本。  
- **性能**：异步、可配置采样间隔、可清理 OCR 索引与摘要。  
- **降级**：OCR 故障不影响 `fact_only` 与 `daily_analysis`。

---

## 八、验收主轴（与里程碑表对齐）

| 主题 | 要点 |
| --- | --- |
| 默认安全 | OCR 与出境摘要默认关 |
| 黑名单 | 命中无 OCR 或可验证 redacted |
| 锚点摘要 | 有数据时出现一行；无数据时不瞎填 |
| 检索 | 仅 P1-1 纳入时验收 FTS |
| 降级 | 关闭 OCR 或模块异常时核心复盘仍可用 |
| AI | 仅双重 opt-in 传出结构化摘要 |
