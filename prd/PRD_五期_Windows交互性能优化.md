# PRD：五期 — Windows 交互性能优化（V5）

**版本**：v1.0 · 2026-04-17  
**上游基准**：`prd/PRD_四期_体验优化与分组管理.md`  
**关联实现**：桌面端 Tauri + React（`project/`）

---

## 1. 背景与问题

当前版本在 macOS 交互流畅，但在 Windows（Win10/Win11）存在以下主观可感知问题：

1. 页面打开后出现掉帧，鼠标移动时存在跟手迟滞。
2. 按钮点击后反馈延迟明显，但功能最终可用。
3. 列表/卡片 hover、弹层打开时卡顿感更突出。

本迭代聚焦**Windows 交互性能专项优化**，保持业务功能与数据口径不变。

---

## 2. 本迭代使命

| 阶段 | 本迭代聚焦 |
|------|------------|
| 一至四期 | 数据底座、智能洞察、正式壳、体验与分组（已完成） |
| **五期** | **Windows 交互流畅度提升：鼠标移动不卡、点击反馈更快、主路径掉帧收敛** |

---

## 3. 用户与场景

### 3.1 主路径场景

- 在 `/lens`、`/timeline`、`/intents`、`/settings` 进行常规浏览与点击操作时，交互流畅无明显卡顿。
- 在会话与时间线列表中快速移动鼠标，hover 反馈稳定，不出现明显“拖影感/发涩感”。
- 点击按钮、打开弹层、切换页签时，反馈时延可接受，不影响连续操作。

### 3.2 平台边界

- 本期优化对象：Windows 10 / Windows 11（WebView2）。
- macOS 不做体验降级，需验证无回归。

## 5. 关键指标与验收阈值

> 本期以“用户主观流畅度”与“可重复的操作脚本”共同验收；指标用于对比，不作为绝对硬件上限承诺。

### 5.1 验收口径（P0）

1. **鼠标移动流畅度**：主路径页面持续移动鼠标 30 秒，无持续性卡顿感。
2. **点击反馈时延**：常见按钮点击到可见状态变化明显优于优化前版本。
3. **主路径稳定性**：不引入功能异常、数据异常和崩溃。
4. **跨平台回归**：macOS 体验不回退，核心流程保持可用。

### 5.2 记录模板（必填）

- 设备信息（CPU/GPU/内存、系统版本、WebView2 版本）
- 构建信息（分支、commit、构建时间、模式）
- 操作脚本（页面、动作序列、持续时间）
- 结果记录（优化前/后主观评分与现象备注）

---

## 6. 方案摘要（实现约束）

1. **平台分支**：仅在 Windows 启用性能优化样式策略。
2. **优先级**：先 CSS/动效降载，再交互链路减负，最后再看壳层参数微调。
3. **低风险改造**：优先做可逆、可控、低侵入修改，避免一次性重构。
4. **可追溯**：所有调整对应到页面路径与验收脚本项。

---

## 7. 验收要点（Given / When / Then 摘要）

1. **Given** Windows 环境打开应用主路径页面，**When** 连续移动鼠标与快速点击，**Then** 无明显掉帧与点击迟滞感。
2. **Given** 时间线与会话列表，**When** 高频 hover 与切换项，**Then** 视觉反馈稳定，无明显卡顿。
3. **Given** 弹层预览与主界面切换，**When** 多次开关弹层，**Then** 交互响应稳定，不卡住主线程。
4. **Given** macOS 环境，**When** 执行同主路径操作，**Then** 功能与体验不回退。
