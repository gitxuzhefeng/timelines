## 执行摘要

**macOS / Windows** 使用 **系统原生、本机离线** OCR：**Vision**、**Windows.Media.Ocr**。**Linux** 使用 **Tesseract**。统一 `**OcrEngine`**（阅读序行、词/行级置信度、可选几何供内部使用）。**禁止**默认或静默 **在线 OCR**。**本期不引入** ONNX/第二推理栈及 **跨引擎自动降级**（详见 OCR 工程方案）。

识别服务 **可检索与情境辅助**；输入为 **已落盘快照**，经 **黑名单与脱敏**；**非全量**（重要触发优先，poll 结合感知哈希、节流、低并发）；失败 **不影响** 事实层与截图主流程。

---

## 1. 产品与技术定位

- OCR 用于：**会话视觉摘要、检索关键词、叙事锚点**。  
- **不写入** `daily_analysis` 既有硬指标公式。

---

## 2. 引擎选型


| 平台      | 引擎                                                     |
| ------- | ------------------------------------------------------ |
| macOS   | Vision（`VNRecognizeTextRequest`），Swift + FFI/sidecar   |
| Windows | `Windows.Media.Ocr`（WinRT，Windows 10+），注意 OCR **语言组件** |
| Linux   | Tesseract（语言包如 `chi_sim+eng`）                          |


Rust 侧 `**OcrEngine`** 统一输出，供闸门与 FTS 前处理共用。

---

## 3. 精度与输出

- 目标：**支撑搜索与粗粒度情境**；须带 **状态或置信度/降级**。  
- 语言与 **系统/用户配置** 对齐；Windows 缺组件时须 **可理解降级**。

---

## 4. 数据来源与闸门

- 输入：**策略允许的已落盘 WebP**；异步解码，**不阻塞**截图写入。  
- **黑名单**命中：**不做 OCR、不写视觉索引**。  
- 与 **录屏权限、AFK、视觉总开关** 一致。

---

## 5. 识别策略（非全量）

高优先级触发优先；poll 与 **感知哈希** 去重；**最小间隔**；**全局并发 1～2**，背压时跳过或延后。

---

## 6. 隐私、脱敏与出境

- 默认可检索内容 **仅存本机**。  
- 出境须 **单独同意**；仅 **脱敏摘要** 等最小必要，与 `PRD_二期AI增强_智能洞察` 一致。  
- **后处理**：乱码清理；手机、证件、卡号、邮箱、IP、密码形态等 **正则脱敏**；用户敏感词。

---

## 7. 可靠性与检索

- OCR **独立队列**，与 capture **解耦**；**超时即失败**，不阻断快照与 `fact_only`。  
- **FTS5**：多词 **AND**；`snippet` 预览；黑名单应用层过滤。  
- **不做** 检索结果在图上的 **像素级字框**（本期）。
