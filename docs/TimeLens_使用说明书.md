## 1. TimeLens 是什么

**一句话**：TimeLens（时间透视镜）是一款**本地优先**的桌面应用，在 **macOS / Windows** 上**被动采集**前台窗口、应用切换与使用时长，配合**智能截图**帮助你在事后还原「那段时间在做什么」，数据默认**只保存在本机**。

### 1.1 能帮你解决什么

| 你可能遇到的困扰 | TimeLens 的做法 |
|------------------|-----------------|
| 忙了一天，却说不出时间花在哪儿 | 会话聚合 + 应用/窗口维度，用数据对齐主观感受 |
| 想回忆「昨天下午三点在干什么」 | 切换即记录，截图辅助情境回溯 |
| 担心监控类工具上传隐私 | **本地优先**，核心数据不上传即可使用 |
| 传统时间追踪要不停点「开始/结束」 | **被动采集**，后台按策略记录，减少手动操作 |

### 1.2 不做什么（边界）

- **不记录**键盘按键内容、剪贴板全文与密码框内容（采集以窗口元数据与策略内截图为主，具体以后台实现与设置页说明为准）。
- 不是云端考勤系统；可选 AI 等功能若开启，请以**设置页**与网络策略说明为准。

## 2. 界面与功能一览

应用打开后，左侧为主导航，顶部可切换**日期**、控制**采集状态**（开始 / 停止采集）。

### 2.1 主导航（主界面）

| 模块 | 路径（路由） | 说明 |
|------|----------------|------|
| **今日透视** | `/lens` | 当日结构、管线摘要等总览（默认进入应用后的主视图之一） |
| **时间线** | `/timeline` | 按时间轴查看会话与片段，便于按时段核对 |
| **日报告** | `/report` | 日维度报告（含事实层与可选 AI 相关能力，以当前版本为准） |
| **应用分组** | `/intents` | 管理应用意图 / 分组，便于统计与复盘维度一致 |
| **设置** | `/settings` | 权限、采集引擎、OCR、AI 等选项 |

### 2.2 开发工具（需在设置中开启「开发模式」）

开启后，侧栏会出现 **Dev** 分组，常见入口包括：

| 模块 | 说明 |
|------|------|
| **复盘（旧入口）** | 历史复盘入口，便于对照新旧流程 |
| **会话** | 查看会话列表与截图等调试信息 |
| **OCR 检索 / OCR 评估** | 与屏幕文字识别检索、质量评估相关 |
| **健康** | 查看各采集/处理引擎的健康状态 |

> 普通使用者可忽略 **Dev** 菜单；开发者排障、验收时建议打开。

### 2.3 顶部「采集状态」

- **采集中**：后台正在按设定进行窗口追踪与（若启用）截图等流程。  
- **停止采集**：暂停采集；需要再次点击以恢复。  
- 首次使用或系统权限不足时，请按 **设置** 页指引在系统中授予辅助功能、屏幕录制等权限（**macOS / Windows 文案与入口可能不同**，以应用内说明为准）。

## 3. 日常使用（终端用户）

### 3.1 安装方式

- **macOS**：使用发布页提供的 **DMG**（或本仓库构建产物 `project/electron-dist/*.dmg`），打开后将应用拖入「应用程序」。若系统提示未识别开发者，可在「系统设置 → 隐私与安全性」中选择仍要打开（未公证的本地构建常见此提示）。  
- **Windows**：使用发布页或构建生成的 **安装程序**（如 NSIS `Setup` 安装包）。安装后从开始菜单或桌面快捷方式启动。

### 3.2 第一次打开建议顺序

1. 打开 **设置**，完成**权限**相关步骤（辅助功能 / 屏幕录制 / 通知等，依平台与版本而定）。  
2. 回到主界面，确认顶部 **采集状态** 为「采集中」（若默认未启动，可手动点击开始）。  
3. 切换到 **今日透视** 或 **时间线**，确认有新会话或事件产生（切换几次前台应用后再查看更易看到效果）。  
4. 需要复盘某一天时，在顶部**日期选择器**中切换到目标日期。

### 3.3 托盘与窗口（Electron 壳）

在 **Electron** 模式下，应用可能提供**系统托盘**（取决于当前版本是否配置图标与平台行为）：

- **macOS**：关闭窗口后应用可仍在 Dock 中运行；可通过托盘菜单或再次点击 Dock 图标显示主窗口（具体以当前构建为准）。  
- **Windows**：关闭所有窗口后通常会退出应用；托盘可快速显示主窗口或退出。

若你使用的是 **Tauri** 发行版，托盘与关闭行为以 Tauri 构建为准，细节可能略有不同。

## 4. 隐私与数据

- 数据默认存储在**本机**（具体目录随平台与用户目录变化，详见设置或文档中的「数据目录」说明）。  
- 核心设计理念是**可审计的本地数据**：你可以在自己的磁盘上备份、删除数据。  
- 若使用 **AI 相关功能**，请在使用前阅读设置中的 **隐私说明 / 端点配置**，确认数据是否离开本机。

## 5. 给 Electron 新手：本项目中 Electron 扮演什么角色

如果你不熟悉 **Electron**，可以把它理解成：**用网页技术（HTML/CSS/JS）做界面，用 Node.js 在「主进程」里访问系统能力，两者通过 IPC 通信**。

### 5.1 三个常用概念

| 概念 | 通俗理解 |
|------|----------|
| **主进程（Main）** | 对应本仓库的 `project/electron/main.mjs`：负责创建窗口、拉起原生侧进程、转发网络或 IPC。 |
| **渲染进程（Renderer）** | 用户看到的界面，即 Vite 构建出来的前端页面（与浏览器标签页类似，但受 Electron 安全策略约束）。 |
| **预加载（Preload）** | `project/electron/preload.cjs`：在隔离环境下向页面注入安全的 API（如 `window.timelensDesktop`），避免页面直接操作 Node（须用 CommonJS，与 `sandbox: true` 兼容）。 |

### 5.2 本项目中多出来的「一块」：Rust 守护进程

在 **Electron** 方案中，业务核心仍由 **Rust**（`project/src-tauri`）实现，但以 **`timelens-daemon`** 形式作为**独立进程**运行：

- 主进程启动 `timelens-daemon`，通过 **本地 HTTP**（如调用指令）与 **WebSocket**（推送事件）与界面协作。  
- 这样前端不必改业务逻辑即可在 **Tauri**（进程内嵌）与 **Electron**（子进程）之间切换架构。

**你作为新手需要记住的**：改界面 → 看 `project/src`；改采集/数据库/系统接口 → 看 `project/src-tauri`；改 Electron 启动方式、托盘、打包 → 看 `project/electron` 与 `package.json` 的 `build` 配置。

## 6. 给项目新成员：仓库结构、双壳与推荐工作流

### 6.1 仓库顶层在做什么

- 多数字典级文档在 **`docs/`**，产品需求在 **`prd/`**。  
- **可运行的桌面应用工程**在 **`project/`** 目录下（前端 + `src-tauri` + Electron 脚本）。

### 6.2 `project/` 里要关心的路径

| 路径 | 含义 |
|------|------|
| `project/src/` | React 界面、路由、状态管理 |
| `project/src-tauri/` | Rust 核心：采集、存储、Tauri 命令、守护进程二进制 |
| `project/electron/` | Electron 主进程、preload |
| `project/package.json` | 脚本入口、`electron-builder` 打包配置 |

### 6.3 两种运行形态：Tauri vs Electron

| 方式 | 典型命令（在 **`project`** 目录下） | 说明 |
|------|--------------------------------------|------|
| **Tauri 开发** | 在仓库根目录：`npm run tauri dev`（根 `package.json` 已配置则可直接用） | Rust 与 WebView 一体化，适合传统 Tauri 迭代 |
| **Electron 开发** | `npm run electron:dev` | 会先编译 `timelens-daemon`，再启动 Vite + Electron |

> 若你从旧文档只看到 `tauri dev`，而当前任务在 Electron 上，请改用 **`project` 下的 `npm run electron:dev`**，并确保本机已安装 **Node.js、Rust、对应平台构建依赖**。

### 6.4 推荐协作流程（新成员）

1. 阅读本说明书与根目录 `README.md` 中的「文档索引」。  
2. 能跑通 **`npm run electron:dev`** 或 **`npm run tauri dev`** 其一（依你负责的分支）。  
3. 改 UI 后用浏览器或 Electron 窗口确认；改 Rust 后用 `cargo test` / 手动跑守护进程验证。  
4. 提交前在目标平台（macOS 或 Windows）做一次完整启动与采集冒烟测试。

## 7. 从源码运行

以下命令默认在 **`project`** 目录执行（请先 `npm install`）。

### 7.1 仅前端（不启动桌面壳）

```bash
npm run dev
```

浏览器访问 Vite 提示的本地地址；注意：**没有 Rust 后端时，多数功能不可用**，仅适合调 UI。

### 7.2 Electron + 守护进程（推荐用于当前 Electron 线）

```bash
npm run electron:dev
```

含义简述：编译 **`timelens-daemon`** → 启动 **Vite（端口以控制台为准，常见 1420）** → 启动 **Electron** 加载该地址。

环境变量 **`TIMELENS_DAEMON_PATH`** 可指向自定义守护进程可执行文件路径（高级用法）。

### 7.3 Tauri（若你负责 Tauri 发行版）

在仓库根目录（视根 `package.json` 脚本而定）：

```bash
cd project && npm install && cd ..
npm run tauri dev
```

具体以根目录 `package.json` 与 **Tauri 官方前置条件**为准；Windows 环境补充说明见 `docs/windows兼容项目/Windows上手.md`。

### 7.4 测试

```bash
npm test
```

会运行 `src-tauri` 下的 Rust 测试（路径以 `package.json` 中 `test` 脚本为准）。

## 8. 打包与分发

以下均在 **`project`** 目录下操作，且需已安装依赖。

### 8.1 打 macOS 包（在 macOS 上执行）

```bash
npm run electron:pack:mac
```

常见产出目录：`project/electron-dist/`（例如 `.dmg`、`.app`）。Apple Silicon 与 Intel 的架构以实际构建日志为准。

### 8.2 打 Windows 包（建议在 Windows 上执行）

```bash
npm run electron:pack:win
```

需本机已用 ** MSVC 工具链** 成功执行 `cargo build --release --bin timelens-daemon`，以生成 **`timelens-daemon.exe`** 并被打进安装包。  
在 macOS 上交叉编译 Windows 原生二进制**成本高且易失败**，团队一般使用 **Windows 实体机 / 虚拟机 / CI Windows Runner** 出 Windows 安装包。

### 8.3 通用打包命令

```bash
npm run electron:pack
```

在当前操作系统上走 `electron-builder` 的默认目标（在 Mac 上多为 mac，在 Windows 上多为 win）。

### 8.4 签名与公证（发布级）

面向公开发布时，macOS 需要 **Developer ID** 与 **公证**，Windows 需要 **代码签名证书** 等。此处不展开；未签名包在本机试用通常可行，但分发时用户可能看到安全警告。

## 9. 常见问题

### 9.1 界面空白或一直加载失败

- 确认 **`timelens-daemon` 已启动**（Electron 模式下主进程应能拉起；若手动调试，可看控制台日志）。  
- 确认 **Vite 地址与 Electron 加载的 URL 一致**（开发模式常见为 `http://127.0.0.1:1420`）。

### 9.2 没有会话 / 没有截图

- 检查 **采集状态** 是否为「采集中」。  
- 检查 **系统权限**（macOS：辅助功能、屏幕录制；Windows：按设置页与系统版本说明操作）。  
- 查看 **设置** 中截图与引擎相关开关。

### 9.3 Electron 与 Tauri 行为不一致

属于预期现象：两者**壳层**不同，但团队目标是在业务层对齐。排障时请说明：**平台（macOS/Windows）+ Tauri 或 Electron + 版本号**。

### 9.4 我从旧分支来，只习惯 `tauri dev`

可以继续在 **`project`** 下使用 Tauri 工作流；若当前主线任务在 Electron，请同时熟悉 **`npm run electron:dev`** 与 **`timelens-daemon`** 日志，避免「界面起来了但后端未连上」。

### 9.5 文档里旧链接失效

仓库经历重构后，部分历史文档在 **`docs/archived/`**。以 **根 `README.md` 文档索引** 与本 **`TimeLens_使用说明书.md`** 为准。
