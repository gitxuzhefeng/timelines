# TimeLens Windows 绿色版打包说明

> **版本**：v1.0 · 2026-03-30  
> **定位**：与安装包共用同一构建产物；发布时以 **ZIP 压缩 bundle 目录** 为主。

---

## 1. 构建

在 Windows 开发机或 CI（`windows-latest`）上，于 `project` 目录执行：

```bash
npm ci
npm run tauri build
```

具体子命令以 `project/package.json` 为准（常见为 `tauri build`）。

---

## 2. 产物位置

构建成功后，可执行文件与依赖通常在：

`project/src-tauri/target/release/bundle/`

目录结构以 **Tauri 2 当前版本** 实际输出为准（可能包含 `nsis` / `msi` 子目录及可运行目录）。**绿色版**取 **可直接运行的应用目录**（含 `.exe`、WebView 资源、DLL 等），整体打包为 ZIP。

---

## 3. 用户使用方式

1. 解压 ZIP 到任意路径（路径含空格时建议一并测试）。  
2. 运行主程序 `TimeLens.exe`（名称以 `tauri.conf.json` 中 `productName` / `mainBinaryName` 为准）。  
3. 数据默认写入 **`%USERPROFILE%\.timelens`**，与 macOS `~/.timelens` 对应。

---

## 4. WebView2

- **Windows 11** 通常已带 Evergreen WebView2。  
- **Windows 10** 若启动白屏或无法显示 UI，请安装 [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)（Evergreen）。  

是否在 ZIP 内捆绑 Fixed Version Runtime 为**后续优化项**，需单独评估体积与许可。

---

## 5. 签名与 SmartScreen

未代码签名的 exe 可能触发 **Windows SmartScreen**。绿色版分发时可在手册中说明「更多信息 → 仍要运行」；企业环境建议后续增加 **Authenticode 签名**（非本绿色版说明必选）。

---

## 6. 与「安装版」关系

| 项 | 绿色版 | 安装版（若后续提供） |
|----|--------|----------------------|
| 二进制 | 相同 | 相同 |
| 分发 | ZIP | NSIS/MSI 等 |
| 代码分支 | 无 | 无 |

---

## 7. 维护

Tauri 或 bundler 升级后，若输出路径变化，请同步更新本节路径描述。
