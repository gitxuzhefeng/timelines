# TimeLens Windows 兼容 — 测试计划

> **版本**：v1.0 · 2026-03-30  
> **范围**：P0 在 **Windows 10 / Windows 11** 上的功能与稳定性验证  
> **分发假设**：绿色 ZIP；数据目录 `%USERPROFILE%\.timelens`

---

## 1. 测试目标

1. 验证 Windows 与 macOS 在 **P0 数据闭环**上行为一致（在平台差异允许范围内）。  
2. 验证 **托盘、权限提示、打开设置、验证面板** 可用。  
3. 记录 **WebView2、多屏、DPI、安全软件** 等环境差异导致的 Known issue。

---

## 2. 测试环境

### 2.1 硬件与系统

| 环境 ID | OS | 说明 |
|---------|-----|------|
| ENV-W10 | Windows 10 22H2（或团队约定版本） | 必测；关注 WebView2 |
| ENV-W11 | Windows 11 | 必测 |

### 2.2 前置条件

- 安装 **WebView2 Evergreen**（若 ENV-W10 未预装，按用户手册安装后重测）。
- **Rust / 构建产物**：与研发交付的 commit 或 CI artifact 一致。
- 测试账号建议使用 **标准用户**（非仅管理员），以贴近真实用户权限。

### 2.3 显示器配置（建议至少覆盖其一）

- 单屏 100% DPI  
- 双屏扩展 + 至少一侧 125%/150% 缩放  

---

## 3. 测试矩阵（P0 对齐）

下列用例映射 `prd/一期_研发任务与验收计划_P0.md` 中 **M2 / M4 / M7** 等核心条目；在 Windows 上 **Given / When / Then** 语义保持一致，步骤按 Windows 操作习惯调整。

### 3.1 M1 数据底座（冒烟）

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M1-01 | 首次启动，无历史数据 | `%USERPROFILE%\.timelens\data\db.sqlite` 创建，应用不崩溃 |
| WIN-M1-02 | 二次启动 | 迁移幂等、可正常打开 DB |

### 3.2 M2 Tracker + 隐私

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M2-01 | 开启采集，单应用前台保持 | `raw_events` 持续有 `poll`，`state_hash` 稳定 |
| WIN-M2-02 | 从应用 A 切到 B | `window_change`、对应 `app_switches`；标题脱敏符合策略 |
| WIN-M2-03 | 浏览器内仅标签标题变化 | 不出现 Session 异常暴增（对照架构 5.6.3） |
| WIN-M2-04 | 空闲超过 AFK 阈值 | `afk_enter`；AFK 期间截图策略与架构一致 |
| WIN-M2-05 | 黑名单/隐私应用 | 脱敏字段符合预期 |
| WIN-M2-06 | 「权限不足」场景（模拟：拒绝/失败） | 不崩溃；UI 与事件可解释（映射到现有字段） |

### 3.3 M3 聚合（抽样）

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M3-01 | A→B 切换后 Session | A 结束时间合理，B 为当前活跃 |
| WIN-M3-02 | AFK 进出 | AFK 前后 Session 边界正确 |
| WIN-M3-03 | 补偿周期 | 无长期 `is_active` 泄漏（在约定观察窗口内） |

### 3.4 M4 Capture

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M4-01 | 窗口切换触发高优先级截图 | `shots` 下有 WebP，`snapshots` 有记录，`session_id` 符合规则 |
| WIN-M4-02 | 去重开启 | 相似帧不重复落盘（或符合产品阈值） |
| WIN-M4-03 | 手动触发截图 | 队列不阻塞、有成品 |
| WIN-M4-04 | 多屏（若环境具备） | 截图区域符合方案约定（主屏/前台所在屏） |

### 3.5 M5 / M6 IPC 与面板

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M5-01 | Sessions / Snapshots / Raw 列表 | 与 DB 一致，`timelens://` 预览可用 |
| WIN-M6-01 | 日期切换、Session 选择 | 无白屏与控制台致命错误 |

### 3.6 M7 权限、托盘、启停

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M7-01 | 托盘「打开主窗口」 | 主窗显示并聚焦 |
| WIN-M7-02 | 托盘「开始/停止采集」 | `tracking_state_changed` 与 DB 行为一致 |
| WIN-M7-03 | 托盘「退出」 | 进程退出，Writer 正常收尾（无库损坏） |
| WIN-M7-04 | `check_permissions` 与顶栏状态 | 与映射语义一致 |
| WIN-M7-05 | 打开设置类按钮 | 打开预期页面或文档，无静默失败 |

### 3.7 M8 存储（抽样）

| 用例 ID | 描述 | 预期 |
|---------|------|------|
| WIN-M8-01 | 存储统计与清理入口 | 数值合理，清理策略不破坏 DB |

## 4. 非功能与兼容性

| 用例 ID | 类型 | 说明 |
|---------|------|------|
| WIN-NF-01 | 启动时间 | 冷启动在可接受范围（团队自定阈值） |
| WIN-NF-02 | 长时间运行 | ≥2h 采集，内存无异常线性增长 |
| WIN-NF-03 | 绿色版解压路径 | 含空格/非 ASCII 路径（可选） |
| WIN-NF-04 | 杀软/Defender | 记录是否误报；不阻塞必测 |

## 5. 回归范围

- **必须**：每次 Windows 相关 PR 合并前，在 **至少一个** Windows 环境跑 **WIN-M1-01、WIN-M2-02、WIN-M4-01、WIN-M7-01～03**。  
- **发版前**：完整跑第 3 节矩阵中标记为 **必测** 的全部用例（ENV-W10 + ENV-W11）。

---

## 6. 通过 / 失败标准

- **通过**：必测用例全部通过，无 P0 级缺陷；非 P0 缺陷已登记并评估可延期。  
- **阻塞**：崩溃、DB 损坏、采集完全无数据、截图始终失败且无提示。
