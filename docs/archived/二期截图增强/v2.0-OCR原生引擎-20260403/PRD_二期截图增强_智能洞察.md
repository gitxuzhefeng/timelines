## 〇、背景与问题

用户在已有「按 App / 会话」复盘的基础上，需要把 **屏幕上的短文本**（窗口标题、票据号、搜索词等）变成 **可检索、可摘要、可选参与 AI 叙事的本地信号**，以便回忆「当时在做什么具体事」。  
**本期不解决**：语义搜索、云端 OCR、修改日终硬指标公式、保证每张截图均可识别。

## 〇、本迭代使命（路线图位置）

在二期 **事实层 + 可选 AI 叙事** 之上，增加 **本地 OCR 增强层**：默认关闭、可感知、可降级，与 AI 出境 **单独 opt-in** 对齐。

## 〇、用户与场景

| 场景 | 主路径 |
| --- | --- |
| 开启 OCR | 设置 → 阅读说明 → 确认 → 打开总开关 |
| 看会话在忙什么 | 会话详情见一行「来自屏幕文字」的摘要；无则看原因说明 |
| 凭关键词找截图 | 会话页 → 输入关键词（可多词 AND）→ 结果列表 + **匹配片段/全文/截图** → 跳转会话/缩略图；可选「仅当前会话」收窄 |
| AI 复盘 | 已 BYOK 且勾选「OCR 摘要可出境」时，AI 请求体可带脱敏 OCR 摘要，**不带原图** |

## 一、本期价值核心

**一句话**：在用户授权与本地优先前提下，把已有截图变成 **可摘要、可检索、可选参与 AI 叙事** 的屏幕文字信号，让复盘从「只在哪个 App」推进到「更可能在做哪件具体事」，并支持凭 **关键词** 找回当时会话。

| 核心 | 用户得到什么 | 边界（本期承诺到哪里） |
| --- | --- | --- |
| **A · 任务锚点** | 会话详情里 **一行摘要**（来自屏幕可见标题/票号等短文本），来源标明「来自屏幕文字」 | 无可靠文字则不展示或 `unknown`，**不编造**；不保证所有界面可识别 |
| **B · 关键词回忆** | **本地** 关键词检索 → **时间 + 会话 + 匹配片段 + 整段 OCR + 截图预览**；可多词 **AND** 联合过滤；结果可 **一键跳转** | 仅 **关键词匹配（FTS）**，不做语义搜索；黑名单与 redacted **不可命中**；**像素级在图上框选匹配字** 需 OCR 几何输出，**本期不做**（见 §2.2.1） |
| **C · AI 最小上下文** | 用户 **单独勾选** 且脱敏通过后，向 `ai_enhanced` 只传 **结构化文字摘要**，默认 **不传原图** | 不改写事实层数字；须符合二期 AI 增强 PRD 出境红线 |

**体验与信任（与 A/B 同级重要）**：用户能感知 OCR **是否在运行**（最近成功处理时间等）；无摘要时 **说明原因**（未识别 / 已 redacted / 已关闭等），避免静默空白。

## 二、本期核心需求

### 2.1 原则

| 原则 | 说明 |
| --- | --- |
| 增强不替代 | 不修改 `daily_analysis` 公式；OCR 为可选层 |
| 文本优于图片 | 出境优先 **结构化 OCR 摘要** |
| 默认本地 | OCR 与索引在本机；出境须 **单独 opt-in** |
| 宁缺勿乱 | 无把握不强行归类；禁止用确定语气编造 |

### 2.2 功能清单与验收要点（本期必达）

| ID | 功能 | 验收要点 |
| --- | --- | --- |
| **S0-1** | OCR 总开关 + 首次说明 | 默认关；开启前独立说明；关闭后停止新 OCR |
| **S0-2** | 黑名单与敏感过滤 | 命中帧不 OCR 或整帧 redacted；**不出境** |
| **S0-3** | 本地 OCR 管线 | 异步处理已采集帧；结果写入本地存储（如 `session_ocr_context` / 等价） |
| **S0-4** | 会话一行摘要（任务锚点） | 有可靠实体/标题则展示一行；无则空或 unknown；展示可关 |
| **S0-5** | 与 AI 增强衔接 | BYOK 且用户勾选「OCR 摘要可出境」时，仅传 **脱敏结构化摘要**；默认不传图 |
| **S0-6** | 本地关键词检索 | OCR 文本本地索引（如 FTS）；关键词搜索 → 时间 + 会话 + 可选缩略图；黑名单/redacted **不可命中** |
| **S0-7** | 检索结果 → 会话/时间轴跳转 | 每条检索结果可打开 **对应会话或时间轴锚点** |
| **S0-8** | OCR 状态可感知 | 可见 OCR 是否启用、**最近成功处理时间**（或等价健康信息）、可选积压提示；失败有 **可理解文案** |
| **S0-9** | 空摘要 / 无 OCR 可解释 | 无可用 OCR 时摘要区说明原因（未识别、redacted、OCR 关闭、引擎异常等），策略与 `unknown` 展示由 UI 统一 |

### 2.2.1 二级图片增强 · OCR 检索结果深化

在 **S0-6 / S0-7** 基础上，强化 **「搜到之后能看见什么」** 与 **多词精准过滤**，仍全部为 **本地 FTS**，不引入语义检索。

| ID | 功能 | 验收要点 |
| --- | --- | --- |
| **S0-10** | 多关键词联合检索 | 支持空格/中英文逗号/分号分隔；**逻辑与（AND）**；单词兼容原行为；上限与单词长度限制见技术方案 |
| **S0-11** | 结果内容展示 | 每条命中展示：**(a)** FTS **匹配片段**（标出命中部分）、**(b)** 该帧 **整段脱敏 OCR 全文**（可截断）、**(c)** **截图预览**；并展示 **时间、应用、窗口标题、场景/Intent（学业或工作分类等，与会话 `intent` 一致）** |
| **S0-12** | 识别结果可视化（文本 + 图片角标） | **(a)** 全文区对 **所有检索词** 做高亮；**(b)** 匹配片段区展示 FTS `snippet` 高亮；**(c)** 截图区域以 **角标/标签** 展示当前检索词（示意）；**不做**依赖 OCR 字级坐标的图像框选（延后） |
| **S0-13** | 会话范围收窄 | 提供「**仅当前会话**」选项：勾选后检索仅在该会话的索引子集内执行，用于多词 + 会话联合精准定位 |

### 2.2.2 下一小版本 · OCR 质量与结构化输出（目标）

**背景**：当前管线以 Tesseract 整图 `stdout` 为主，易出现 **中文界面误识、乱码、噪声符号**；摘要取首条有效行时 **未区分置信度**，易把垃圾行写入 FTS 与「来自屏幕文字」摘要。

**产品目标**：在 **仍默认本地、不默认上云** 前提下，提升 **可检索文本可信度** 与 **展示上的行级可读性**，并与 §2.5「像素级框选仍不做」兼容（本版本可为后续字级几何 **预留数据结构**，不承诺 UI 框选）。

| ID | 功能 | 验收要点 |
| --- | --- | --- |
| **S0-14** | 语言与页面分割策略 | **默认**支持 **中文简体 + 英文** 混合识别路径（依赖用户本机已安装对应 `traineddata`，未安装时降级文案与现有「引擎异常」一致）；提供 **PSM（页面分割模式）** 的可配置项（环境变量或设置页高级项，具体入口与产品一致）；在固定 **黄金截图集**（含深浅色、浏览器/IDE/IM）上相对 v4.2 基线 **有效字符比例 / 人工可接受摘要率** 有可测提升（研发自证 + 抽样签字） |
| **S0-15** | 按视觉行排版与存储 | OCR 输出按 **阅读顺序、逐行** 落库与展示（行内词序与横向位置一致，行间按纵向排序）；会话详情「整段 OCR」、检索结果全文区 **换行与视觉行一致**；不改变黑名单、脱敏与 **FTS 不可命中 redacted** 的规则 |
| **S0-16** | 质量闸门（置信度 + 规则） | 利用引擎输出的 **字/词级置信度**（如 Tesseract TSV）做聚合：过滤 **低置信度片段、明显乱码行**（符号占比过高等启发式可与置信度组合）；**不可靠内容默认不进入 FTS**（或统一降权策略在技术方案中写明且可配置）；**一行摘要** 仅从 **通过闸门后的行** 中选取，若无则沿用现有 **空因说明**（未识别 / redacted / 关闭等），**不编造** |
| **S0-17** | 截图轻量预处理（可选） | 提供 **可选** 预处理策略（如缩放至合适字高、灰度/对比度/自适应二值化等 **至少一种** 可开关路径），改善小字、深色模式、复杂背景；默认行为以 **不拖垮异步队列** 为准，超时/失败回退为当前无预处理路径 |

**工程配套**：实现细节见 **`prd/二期截图增强_OCR专题优化技术方案.md`（v2.1）**；执行摘要见 `prd/二期截图增强_技术方案.md`。本节只锁定 **交付边界与验收口径**。

### 2.3 非功能约束（摘要）

- **隐私**：黑名单为第一道；OCR 文本第二道敏感模式扫描；日志不得包含完整屏幕文字。  
- **性能**：异步、可配置采样、可清理 OCR 索引与摘要。  
- **降级**：OCR 异常不影响 `fact_only` 与 `daily_analysis`。

### 2.5 本迭代不做（明确延后）

| 项 | 说明 |
| --- | --- |
| 语义 / 向量检索 | 仅关键词（FTS） |
| 默认云 OCR | 不接默认商业云 OCR |
| 改动 `daily_analysis` 公式 | 禁止；视觉进指标须另立项 + 指标字典 |
| 系统原生 OCR | **桌面**为 macOS Vision、Windows.Media.Ocr；**Linux** 为 Tesseract；**本期不实现** ONNX 与跨引擎自动降级；详见 `prd/二期截图增强_OCR专题优化技术方案.md` v2.1、`rule/TimeLens_TechArch_Phase2_截图增强.md` v2.1 |
| 像素级「在图上框出命中文字」 | 依赖 OCR 引擎输出 **字/词级几何**（如 Tesseract TSV、hOCR 或系统 Vision 框）；**不纳入本期** |

### 2.6 验收原则

与 `二期截图增强_里程碑与验收计划.md` 中 **Given / When /Then** 同源；**v4.2** P0 需求 S0-1～S0-13 均可在该表或接口契约中找到对应用例或字段；**下一小版本** S0-14～S0-17 在该里程碑文档中 **增补对应用例** 后生效。

### 2.7 降级与默认行为

| 条件 | 用户体验 |
| --- | --- |
| OCR 关闭 | 无新 OCR；摘要区说明「OCR 已关闭」 |
| 引擎不可用 / 超时 | 不阻塞截图；状态区可感知失败文案 |
| 黑名单 / 整帧 redacted | 不写入可检索索引；摘要说明原因 |
| AI 未勾选 OCR 出境 | 请求体不含 OCR 摘要，仅原有聚合 JSON |
