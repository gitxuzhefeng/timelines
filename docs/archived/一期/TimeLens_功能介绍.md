# TimeLens 功能介绍

> **AI 驱动的个人时间透视镜**
>
> 自动、被动地记录你的电脑使用行为，用客观数据帮你看清时间真正流向了哪里。

---

## 核心价值主张

### 为什么需要 TimeLens？

现代知识工作者每天在电脑前工作 8 小时以上，却对"这 8 小时真正发生了什么"几乎一无所知。

**TimeLens 解决的核心问题**：
- ❌ 主观感受与实际产出脱节
- ❌ 不知道时间浪费在哪里
- ❌ 无法量化工作效率
- ❌ 缺乏客观的工作记录

**TimeLens 提供的解决方案**：
- ✅ 被动采集，零操作负担
- ✅ 客观数据，打破认知盲区
- ✅ 情境回溯，精确定位工作内容
- ✅ 本地存储，隐私第一

---

## 核心功能

### 1. 被动行为追踪

**自动记录，无需手动操作**

- 每 2 秒检测窗口状态变化
- 自动识别应用切换
- 智能聚合工作会话
- 后台静默运行，不打扰工作

**采集内容**：
- 窗口标题（如"main.rs - VSCode"）
- 应用名称（如"Google Chrome"）
- 切换时间戳
- 持续时长

### 2. 智能截图系统

**自动截图 + 去重优化**

- 窗口切换时自动截图
- pHash 算法智能去重
- WebP 格式压缩存储
- 每日仅占用 5-10MB 空间

**应用场景**：
- "昨天下午 3 点我在看什么文档？"
- "上周那个 Bug 的错误信息是什么？"
- "客户要求的设计稿在哪个版本？"

### 3. 工作会话聚合

**自动识别连续工作时段**

- 将碎片化事件聚合为有意义的会话
- 按应用、项目、任务分组
- 计算真实工作时长
- 识别上下文切换模式

**数据洞察**：
- 今日在 VSCode 工作了 3.5 小时
- Chrome 使用时长 2.1 小时（其中 GitHub 1.2h）
- 应用切换 47 次，平均每 10 分钟切换一次

### 4. 实时状态监控

**系统托盘常驻，一键查看**

- 引擎运行状态
- 今日采集统计
- 存储空间占用
- 快速暂停/恢复

## 技术特性

### 本地优先架构

```
100% 本地运行
    ↓
零网络请求
    ↓
数据完全掌控
    ↓
隐私绝对安全
```

### 高性能设计

- **低资源占用**: 内存 < 100MB，CPU < 2%
- **智能去重**: pHash 算法避免重复截图
- **增量存储**: 每日仅增长 5-10MB
- **异步架构**: 不影响系统性能

### 隐私保护机制

| 采集内容 | 说明 |
|---------|------|
| ✅ 窗口标题 | 用于识别工作内容 |
| ✅ 应用名称 | 用于统计使用时长 |
| ✅ 屏幕截图 | 用于情境回溯 |
| ❌ 按键内容 | **不记录** |
| ❌ 剪贴板 | **不记录** |
| ❌ 密码 | **不记录** |

## 使用场景

### 场景 1：开发工程师

**痛点**: 写了一天代码，说不清楚做了什么

**TimeLens 解决方案**:
- 精确记录每个文件的编辑时长
- 截图保存关键代码片段
- 还原 Git 分支切换历史
- 量化调试时间 vs 编码时间

### 场景 2：自由职业者

**痛点**: 向客户汇报工时无客观依据

**TimeLens 解决方案**:
- 自动生成工时记录
- 截图作为工作证明
- 按项目统计时长
- 导出工时报告

### 场景 3：效率优化者

**痛点**: 感觉效率不高但找不到问题

**TimeLens 解决方案**:
- 识别时间黑洞（社交媒体、邮件）
- 量化上下文切换成本
- 对比主观感受与客观数据
- 发现最佳工作时段

## 技术架构亮点

### 双层数据模型

```
原始事件层 (raw_events)
    ↓ 实时聚合
会话层 (window_sessions)
    ↓ AI 分析（二期）
洞察层 (insights)
```

### 统一写入架构

```
多个采集引擎
    ↓
Writer Actor (单线程队列)
    ↓
SQLite (批量事务)
    ↓
高性能 + 数据一致性
```
