## 一、自动化测试（`cargo test`）

在仓库根目录执行：

```bash
npm run test
# 或
cargo test --manifest-path project/src-tauri/Cargo.toml
```

### 测试用例与 PRD 映射

| 测试（Rust 模块） | 覆盖意图 | PRD 条目（支撑/间接） |
|-------------------|----------|------------------------|
| `migrations::tests::migrations_run_twice_idempotent` | 迁移可重复执行、P0 表存在 | M1-01、M1-02 |
| `writer::tests::apply_batch_raw_inserts_row_and_app_meta` | Writer 单批写入 raw + app_meta | M1-03 |
| `writer::tests::apply_batch_session_insert_and_close` | Session 插入与闭合 | M3 数据形态 |
| `writer::tests::retention_removes_old_snapshot_file_and_clears_path` | 保留策略删文件 + `file_path` 置空 | M8-04 |
| `writer::tests::writer_thread_shutdown_persists_queued_raw` | Writer 线程 + `Shutdown` 后数据可重开读取 | M1-04 |
| `privacy::tests::*` | 标题/Bundle 脱敏与 URL 截断 | M2-05 |
| `aggregation::pipeline::tests::same_app_*` | 同应用判定（标题变不切 Session 的辅助逻辑） | M3-02 |

**说明**：Tracker/Capture/IPC/托盘等依赖**本机权限与 GUI** 的路径无法在纯 `cargo test` 中覆盖，需按第二节手动验收。

## 二、手动主路径 E2E（M9-01）

**环境**：macOS，开发或测试包；建议先清空或备份 `~/.timelens` 以免干扰。

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 首次启动 `npm run tauri dev` 或安装包 | 生成 `~/.timelens/data/db.sqlite`，应用不崩溃 |
| 2 | 系统设置中授予**辅助功能**、**屏幕录制** | 验证面板顶部权限为绿；可点「刷新权限」 |
| 3 | 点击「开始采集」 | 状态为运行中；`raw_events` 开始增长（可配合「全部刷新」） |
| 4 | 切换前台应用到另一 App | 出现 `window_change` / Session 更新；有截图时 `shots/` 与 `snapshots` 一致 |
| 5 | 在验证面板选 Session、看预览 | `timelens://localhost/snapshot/{id}` 能显示 WebP（无外网请求） |
| 6 | 点击「停止采集」 | 状态停止；无 P0 级崩溃 |

**未授权用例**（M2-06 / M7）：在未勾选辅助功能或屏幕录制时启动采集，应不崩溃，权限区为红并有打开系统设置入口。

## 四、P0 范围对照（M9-03）

一期 P0 以 PRD 与架构 **P0 清单**为准；本仓库**不包含** P1（Input Dynamics、Clipboard、settings、配置面板扩展等）。

发布前建议产品/接口 Owner 确认：

- [ ] Command / Event 命名与前端 `src/services/tauri.ts`、`src/types` 一致。  
- [ ] 与 PRD 冲突项（如 AppSwitch 仅 `window_change`）已在实现或注释中落地。  
- [ ] 已知限制已书面备案（例如未签名二进制在部分环境下的权限限制）。

## 五、回归建议

- 发版前：至少跑通 **「一、自动化」** + **「二、手动」** 全路径。  
- 修改 DB / Writer / 迁移后：必跑 `cargo test`。  
- 修改 Capture / 权限后：在**真实权限矩阵**下重复 M4 / M7 相关行。
