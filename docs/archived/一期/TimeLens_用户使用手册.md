## 产品简介

**TimeLens** 是一款 AI 驱动的个人时间透视镜，运行在 macOS 上的工作行为追踪与智能分析工具。

### 核心特性

- ✅ **被动采集**: 自动记录窗口切换、应用使用、工作状态
- ✅ **智能截图**: 自动截取工作场景，支持情境回溯
- ✅ **本地优先**: 100% 数据本地存储，零网络请求
- ✅ **隐私保护**: 不记录按键内容、密码等敏感信息
- ✅ **实时监控**: 系统托盘实时显示工作状态

### 适用人群

- 开发工程师：精确还原每段代码的编写时长
- UI/设计师：记录设计工具和参考资料的使用分布
- 产品经理：量化会议、文档、沟通的时间消耗
- 自由职业者：生成客观工时记录用于计费
- 知识工作者：打破时间使用的认知盲区

## 系统要求

### 最低配置

- **操作系统**: macOS 12.0 (Monterey) 或更高版本
- **处理器**: Apple Silicon (M1/M2/M3) 或 Intel Core i5
- **内存**: 4GB RAM
- **存储空间**: 500MB 可用空间（初始安装）+ 每日约 5-10MB 数据增长
- **权限**: 辅助功能权限、屏幕录制权限

### 推荐配置

- **操作系统**: macOS 14.0 (Sonoma) 或更高版本
- **处理器**: Apple Silicon (M2/M3)
- **内存**: 8GB RAM
- **存储空间**: 2GB 可用空间

## 安装指南

### 方式一：DMG 安装包（推荐）

1. 下载 `TimeLens-v1.0.dmg` 安装包
2. 双击打开 DMG 文件
3. 将 TimeLens 图标拖拽到 Applications 文件夹
4. 在 Launchpad 或 Applications 文件夹中找到 TimeLens
5. 双击启动应用

### 方式二：从源码构建

```bash
# 克隆仓库
git clone https://github.com/your-org/timelens.git
cd timelens

# 安装依赖
npm install

# 构建应用
npm run tauri build

# 安装包位置
# src-tauri/target/release/bundle/dmg/TimeLens_1.0.0_aarch64.dmg
```

## 首次启动

### 1. 权限授予

首次启动 TimeLens 时，系统会要求授予以下权限：

#### 辅助功能权限（必需）
- **用途**: 读取当前活动窗口信息
- **授予方式**:
  1. 打开"系统设置" → "隐私与安全性" → "辅助功能"
  2. 点击左下角锁图标解锁
  3. 找到 TimeLens，勾选启用

#### 屏幕录制权限（必需）
- **用途**: 截取工作场景用于情境回溯
- **授予方式**:
  1. 打开"系统设置" → "隐私与安全性" → "屏幕录制"
  2. 点击左下角锁图标解锁
  3. 找到 TimeLens，勾选启用

### 2. 初始化设置

权限授予后，TimeLens 会自动：
- 创建本地数据库 `~/.timelens/data/db.sqlite`
- 创建截图存储目录 `~/.timelens/data/shots/`
- 启动后台采集引擎
- 在系统托盘显示图标

## 核心功能

### 1. 系统托盘

TimeLens 常驻在 macOS 系统托盘，提供快速访问：

- **状态指示器**: 实时显示采集状态（运行中/暂停/错误）
- **快速菜单**:
  - 打开主界面
  - 暂停/恢复采集
  - 查看今日统计
  - 退出应用

### 2. 主界面

#### 状态监控面板
- **引擎状态**: 显示 Tracker、Capture、Aggregation 引擎运行状态
- **权限检测**: 实时检测辅助功能和屏幕录制权限状态
- **数据统计**: 今日采集事件数、截图数、存储占用

#### Session 列表
- **时间轴视图**: 按时间顺序展示工作会话
- **应用分组**: 按应用名称分组显示使用时长
- **详情查看**: 点击查看单个 Session 的详细信息
  - 开始/结束时间
  - 持续时长
  - 窗口标题
  - 关联截图

#### 截图预览
- **网格布局**: 缩略图网格展示所有截图
- **时间筛选**: 按日期、时间段筛选截图
- **快速查看**: 点击放大查看完整截图
- **情境回溯**: 通过截图快速定位"当时在做什么"

### 3. 数据采集

#### 自动采集内容
- **窗口事件**: 每 2 秒检测窗口切换和状态变化
- **智能截图**: 窗口切换时自动截图（去重优化）
- **应用切换**: 记录应用间切换的有向图
- **工作会话**: 自动聚合连续工作时段

#### 数据存储
- **本地数据库**: `~/.timelens/data/db.sqlite`
- **截图文件**: `~/.timelens/data/shots/` (WebP 格式)
- **存储优化**: pHash 去重，每日约 5-10MB 增长

## 常见问题

### Q1: TimeLens 会记录我的按键内容吗？
**A**: 不会。TimeLens 只记录窗口标题、应用名称、截图等元数据，不记录任何按键内容、剪贴板文本或密码。

### Q2: 数据会上传到云端吗？
**A**: 不会。TimeLens 100% 本地运行，所有数据只存储在您的 Mac 上，不会发送任何网络请求。

### Q3: 如何暂停采集？
**A**: 点击系统托盘的 TimeLens 图标，选择"暂停采集"。再次点击可恢复。

### Q4: 截图会占用多少存储空间？
**A**: 使用 WebP 压缩和 pHash 去重，每日约 5-10MB。可在设置中配置自动清理策略。

### Q5: 如何卸载 TimeLens？
**A**:
1. 退出 TimeLens 应用
2. 将 `/Applications/TimeLens.app` 移到废纸篓
3. 删除数据目录 `~/.timelens/`（可选）

### Q6: 为什么需要辅助功能权限？
**A**: 辅助功能权限用于读取当前活动窗口的标题和应用名称，这是 macOS 系统的安全限制。

### Q7: 截图会包含敏感信息吗？
**A**: 截图会捕获当前屏幕内容。建议在处理敏感信息时暂停采集，或在设置中配置应用黑名单。

## 隐私说明

### 数据收集范围

TimeLens 采集以下数据：
- ✅ 窗口标题（如"文档.docx - Microsoft Word"）
- ✅ 应用名称（如"Google Chrome"）
- ✅ 屏幕截图（WebP 格式）
- ✅ 窗口切换时间戳
- ✅ 应用使用时长

TimeLens **不会**采集：
- ❌ 按键内容
- ❌ 剪贴板文本
- ❌ 密码或敏感表单
- ❌ 网络流量
- ❌ 文件内容

### 数据存储

- **位置**: `~/.timelens/` 目录
- **格式**: SQLite 数据库 + WebP 图片
- **权限**: 仅当前用户可访问
- **备份**: 建议定期备份 `~/.timelens/` 目录

### 隐私保护措施

1. **本地优先**: 所有数据只存储在本地，不上传云端
2. **零网络**: 应用不发起任何外部网络请求
3. **用户控制**: 可随时暂停采集、删除数据
4. **黑名单机制**: 可配置不采集特定应用的数据
