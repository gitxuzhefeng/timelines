# TimeLens 一期（P0）产品研发任务计划表

> **范围**：仅 **P0 核心数据闭环**，**不包含** Input Dynamics、Clipboard Flow、settings 表、配置面板、基础统计页等 P1 内容（以 `TimeLens_TechArch_V2.md` 为准）。  
> **用途**：研发分工、排期、依赖对齐；测试同事按 **Given / When / Then** 勾选验收。  
> **版本**：v1.0 · 2026-03-27

---

## 一、角色说明

| 角色 | 职责摘要 |
|------|----------|
| **后端** | Rust/Tauri：引擎、Writer、DB、迁移、IPC、权限、托盘、文件与截图流水线 |
| **前端** | React：验证面板、Zustand、Virtuoso、IPC 封装、事件订阅、`timelens://` 展示 |
| **测试** | 用例设计、按条目验收、回归清单、缺陷分级与跟踪；**不参与**代码实现 |
| **产品**（可选） | 范围确认、验收拍板、与架构文档冲突时裁决 |
| **接口契约 Owner**（建议 1 人兼任） | Command/Event 命名与字段前后端一致，变更同步 |

## 二、模块依赖总览（必须先读后做）

```
M1 数据底座（库表 + 迁移 + Writer）
        │
        ├──────────────────┐
        ▼                  ▼
M2 Tracker+隐私            M4 Capture（可与 M3 并行开发，集成阶段依赖 M3 的 Session）
        │                  │
        ▼                  │
M3 聚合（Session + AppSwitch + 补偿）◄── 依赖 M2 的 raw 与触发语义
        │
        ├──────────────────┘
        ▼
M5 IPC（P0 Commands/Events 收口 + timelens://）
        │
        ▼
M6 前端验证面板（依赖 M5）
        │
M7 权限 + 托盘 + 采集启停（可与 M2～M4 并行，全链路联调在 M5 后）
        │
        ▼

        ▼
```

| M4 | M1；与 M3 **联调**时需 Session ID 归属规则已定 |
| M5 | M2/M3/M4 能力已具备或可 mock 后逐条替换为真实现 |
| M6 | M5 |
| M7 | M1；与 M2/M4 **联调**时需真实权限与截图 |
| M8 | M1、M4（文件路径）、M3（会话量） |
| M9 | M1～M8 |

## 三、分模块任务与验收（Given / When / Then）

以下条目测试同事**按序号勾选**；失败需记缺陷并标明阻塞模块。

---

### M1 · 数据底座：迁移、双连接、Writer Actor、核心表

**目标**：`schema_migrations` 管理；WAL；写连接仅 Writer；读连接只读；P0 表可创建；`WriteEvent` 全路径可走通。

| 角色 | 任务 |
|------|------|
| **后端** | 实现 `migrations` 与初始 Schema（`raw_events`、`window_sessions`、`snapshots`、`app_switches`、`app_meta`、`schema_migrations`）；读写连接策略；Writer 线程、批量事务、drain、`Shutdown`、基础指标；`rusqlite` 版本按架构升级 |
| **前端** | 无（或仅类型占位，可与 M5 合并） |
| **测试** | 见下方验收表；关注迁移幂等、崩溃恢复后 DB 可打开 |

#### 验收条目（Given / When / Then）

- [ ] **M1-01**  
  - **Given** 首次安装、无 `db.sqlite`  
  - **When** 启动应用  
  - **Then** 生成 `~/.timelens/data/db.sqlite`，`schema_migrations` 记录版本与 P0 表存在，应用不崩溃  

- [ ] **M1-02**  
  - **Given** 已存在旧版本迁移记录  
  - **When** 升级到新迁移版本  
  - **Then** 仅执行增量迁移，且启动成功  

- [ ] **M1-03**  
  - **Given** Writer 收到多条 `WriteEvent`  
  - **When** 同一批事务提交  
  - **Then** 库中记录条数与预期一致，且无多写者竞争崩溃（可由集成测试或日志断言）  

- [ ] **M1-04**  
  - **Given** 应用正常退出流程  
  - **When** 触发 Writer `Shutdown`  
  - **Then** 队列内事件处理完毕或明确丢弃策略有日志，DB 文件无损坏（可重开）

### M2 · Tracker + 隐私：轮询、AFK、RawEvent、AppSwitch 投递

**目标**：2s 轮询；状态机；隐私过滤；`state_hash`/trigger；微抖动；AFK 行为符合架构 5.6；仅通过 Writer 写入。

| 角色 | 任务 |
|------|------|
| **后端** | `tracker.rs`；`privacy.rs`；`utils/hash`、`utils/regex`；向 Writer 投递 `RawEvent`/`AppSwitch`；向 Capture 发信号、向 Aggregation 发命令；`emit` 所需前端事件的数据准备 |
| **前端** | 无 |
| **测试** | 见下方；需在**已授权辅助功能**环境下测；缺权限用例单独成组 |

#### 验收条目

- [ ] **M2-01**  
  - **Given** 已授予辅助功能权限，采集已启动  
  - **When** 前台仅保持一个应用窗口不变超过数个轮询周期  
  - **Then** `raw_events` 持续追加，`trigger_type` 主要为 `poll`，`state_hash` 稳定  

- [ ] **M2-02**  
  - **Given** 采集已启动  
  - **When** 用户从应用 A 切换到应用 B（前台变化）  
  - **Then** 出现 `window_change` 与对应 `app_switches`（简版规则与 PRD/架构一致），且敏感 title 经脱敏策略处理  

- [ ] **M2-03**  
  - **Given** 采集已启动  
  - **When** 同一应用内仅切换浏览器标签（标题变、应用不变，按架构不切 Session）  
  - **Then** `raw_events` 中可出现 `title_change` 或等价记录，**且不应**仅因此产生错误级别的 Session 爆炸（对照架构 5.6.3）  

- [ ] **M2-04**  
  - **Given** 可模拟空闲（或等待真实 AFK 阈值）  
  - **When** `idle_seconds` 超过配置的 AFK 阈值  
  - **Then** 记录 `afk_enter`，且 AFK 期间行为符合架构（不产生多余截图见 M4 联动）  

- [ ] **M2-05**  
  - **Given** 黑名单或隐私策略中的应用/标题  
  - **When** 轮询采集  
  - **Then** 落库字段为脱敏/遮蔽后的结果，**不**落按键与剪贴板正文  

- [ ] **M2-06**  
  - **Given** 未授予辅助功能权限  
  - **When** 启动采集  
  - **Then** 行为可预期（不崩溃、有明确状态），与 M7 权限文案一致  

---

### M3 · 聚合管道：window_sessions、补偿、与 Raw 语义一致

**目标**：按架构 5.6.3 / 5.6.4 折叠 Session；`app_switch` 与 Session 关系正确；补偿任务闭合僵尸 Session；可选：重放工具（不阻塞 P0 若排期紧，但语义必须一致）。

| 角色 | 任务 |
|------|------|
| **后端** | `aggregation/pipeline.rs`；`SessionUpdate` 写入；60s 补偿；与 Tracker 的 cmd 通道对齐 |
| **前端** | 无 |
| **测试** | 重点跨场景：切应用、AFK、长时间单应用、异常退出恢复 |

#### 验收条目

- [ ] **M3-01**  
  - **Given** 已从 A 切到 B  
  - **When** 聚合完成  
  - **Then** A 的 Session `end_time` 合理，B 新 Session `is_active` 与当前前台一致  

- [ ] **M3-02**  
  - **Given** 同应用内标题多次变化（不切 Session 策略）  
  - **When** 查询 `window_sessions`  
  - **Then** Session 条数不因标题抖动而异常暴增（对照架构说明）  

- [ ] **M3-03**  
  - **Given** AFK 进入  
  - **When** AFK 前后  
  - **Then** AFK 前 Session 已结束；AFK 退出后新建 Session（即使回到同一应用）  

- [ ] **M3-04**  
  - **Given** 某 Session 长时间无新 raw 归属且非 AFK  
  - **When** 补偿周期运行  
  - **Then** Session 被闭合或按架构规则修正，无永久 `is_active=1` 泄漏（在合理测试窗口内）

### M4 · Capture：截图流水线、WebP、pHash 去重、snapshots

**目标**：接收 `CaptureSignal`；screencapture；缩放编码；按策略去重；`WriteEvent::Snapshot`；文件落盘路径规范；与 Session 归属（信号时刻 Session ID）一致。

| 角色 | 任务 |
|------|------|
| **后端** | `capture.rs`；阻塞池编码；`timelens://` 或等价本地 asset 映射；`trigger_screenshot` |
| **前端** | 预览区依赖 M6，此处仅联调接口 |
| **测试** | 需**屏幕录制**权限；覆盖高/低优先级信号、去重、手动截图 |

#### 验收条目

- [ ] **M4-01**  
  - **Given** 已授予屏幕录制权限  
  - **When** 发生窗口切换触发高优先级截图  
  - **Then** `shots/` 下生成 WebP，`snapshots` 表有记录，且 `session_id` 与规则 5.6.5 一致  

- [ ] **M4-02**  
  - **Given** 去重开启  
  - **When** 连续 poll 驱动截图画面几乎不变  
  - **Then** 跳过存储次数符合 pHash 阈值设计（或文档说明行为）  

- [ ] **M4-03**  
  - **Given** AFK 状态  
  - **When** 等待  
  - **Then** 不产生有效截图（与架构 AFK 不截图一致）  

- [ ] **M4-04**  
  - **Given** 用户点击手动截图（若 UI 在 M6）  
  - **When** 调用 `trigger_screenshot`  
  - **Then** 生成文件与元数据，且不崩溃  

- [ ] **M4-05**  
  - **Given** 未授予屏幕录制权限  
  - **When** 应截图时  
  - **Then** 失败可观测、应用稳定，与权限模块一致

### M5 · IPC：P0 Commands / Events / timelens://

**目标**：架构 §6.2 P0 Commands 与 §6.3 P0 Events 全部实现且命名稳定；前端可 `invoke`/`listen`；截图 URL 可加载。

| 角色 | 任务 |
|------|------|
| **后端** | `api/commands.rs` 全量；`emit` 频率与 payload 类型；注册 asset 协议 |
| **前端** | `services/tauri.ts`、`types/index.ts`、与后端字段对齐 |
| **测试** | 可基于控制台或最小页面验证 Command 返回值与 Event 触发；全量 UI 在 M6 做 |

#### 验收条目

- [ ] **M5-01**  
  - **Given** 应用已启动  
  - **When** 依次调用 `start_tracking` / `stop_tracking` / `is_tracking`  
  - **Then** 状态一致，且 `tracking_state_changed`（若已订阅）符合  

- [ ] **M5-02**  
  - **Given** 库中已有 sessions/snapshots  
  - **When** 调用 `get_sessions`、`get_session_snapshots`、`get_app_switches`、`get_raw_events_recent`（limit）、`get_activity_stats`（若 P0 已实现）、`get_all_app_meta`  
  - **Then** 返回结构与类型正确，无未处理异常  

- [ ] **M5-03**  
  - **Given** Writer 在工作  
  - **When** 调用 `get_writer_stats`  
  - **Then** 指标非空且合理（批次、耗时等）  

- [ ] **M5-04**  
  - **Given** 某张截图文件已存在  
  - **When** 前端使用 `timelens://` 对应 URL 加载  
  - **Then** 图像可显示（无 CORS 类错误）  

- [ ] **M5-05**  
  - **Given** 采集过程中  
  - **When** Session 更新、保存新截图、AFK、权限变化、writer 指标刷新  
  - **Then** 对应 Event 在预期时机触发（允许与文档频率略有差异但需文档备注）

### M9 · 整体验收、回归与性能基线

**目标**：跨模块 E2E；稳定性；文档 §10.3 性能目标抽样验证（不要求实验室级，需记录环境）。

| 角色 | 任务 |
|------|------|

#### 验收条目

- [ ] **M9-01**  
  - **Given** 完整安装包

- [ ] **M9-03**  
  - **Given** 一期范围冻结  
  - **When** 对照架构 P0 清单  
  - **Then** 无未交付的 P0 项或已书面备案
