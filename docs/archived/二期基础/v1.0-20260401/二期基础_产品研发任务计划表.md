## 二、里程碑依赖（不可颠倒）

```
M0 ─────────┬─→ M1（指标 A+B）──┬─→ M3（报告+导出）──→ M5-校准（Intent+黑名单）
            │                   │              ↑
            └─→ M2（指标 C+D）──┘              │
                                M4（最小 UI）──┘
```

| 工作包进入条件 | 说明 |
| --- | --- |
| M1、M2 | M0 相关任务包 **全部完成** |
| M3 | M1、M2 **全部完成**（`daily_analysis` P0 字段齐） |
| M4 | M3 报告与导出 **可用**（可先并行 UI 骨架，见下） |
| M5-校准 | M3 + M4 **全部完成** |
| DG 降级矩阵 | **M3 完成后** 集中跑（可与 M4 穿插，但须在发布前闭环） |

**可并行**：M1 ∥ M2；M4 的**路由/空壳页面**可在 M1 完成后启动，**真实数据联调**须在 M3 完成后。

## 四、建议执行顺序（拓扑）

按下面块顺序排迭代；块内序号可微调，**不得违反第三节前置列**。

1. **M0-W1 → W7**（顺序：W1 → W2/W3/W4/W5 可部分并行 → W6 → W7）  
2. **M1-W1、M1-W2** 并行 → **M1-W3**  
3. **M2-W1、M2-W2** 并行 → **M2-W3**  
4. **M3-W1 → W2 → W3**  
5. **M4**：W1 尽早；**W2～W5** 在 M3-W3 后集中联调  
6. **M5C-W1**（校准）  
7. **DG-X** → **REL-X**
