## 一、二期 v1 唯一主线

### 1.1 一句话

**交付一份用户每天都想看的、数字可溯源的工作复盘报告**——关闭 AI（本版即无 AI）时功能完整、体验可用。

### 1.2 核心需求（北极星）

| 需求 | 用户语言 | 二期 v1 如何满足 |
| --- | --- | --- |
| **可见** | 今天时间花在哪儿？ | 日终报告：时间分布、应用占比、深度工作段 |
| **可信** | 数据是真的吗？ | 数字来自 `daily_analysis`，可对 SQL；非 AI 模板 |
| **可找回** | 某时刻在干什么？ | 会话列表 + 截图时间对齐 |
| **可校准** | 分类不对 / 某 App 不想统计 | Intent 纠错 + `intent_mapping`；应用黑名单（含采集侧过滤） |

「可叙述」中的 **自然语言叙事** 划归 **二期 AI 增强**；v1 以 **模板化事实摘要** 满足「读得懂」。

### 1.3 三件核心事情（v1）

```
事情 1：补齐数据管道（M0）
事情 2：本地分析引擎 + 事实层日终报告 + 导出（M1～M3）
事情 3：最小可用 UI + 校准闭环（M4 + M5-校准）
```

## 二、事情 1：补齐数据管道（M0）

与合并版 PRD **§二** 一致：`raw_events` / `window_sessions` / `snapshots` / `app_switches` 持续写数；`input_metrics`、`clipboard_flows`、`notifications`、`ambient_context` 按架构补齐；**降级策略** 见合并版 §2.3；**`get_pipeline_health`** 六引擎状态。

**说明**：各引擎在 macOS / Windows 上的实现差异、启发式通知等 **以实际架构与里程碑验收为准**，不阻塞 v1 将「事实层 + 降级标注」定义为交付。

---

## 三、事情 2：本地分析引擎 + 日终复盘报告（M1～M3）

### 3.1 架构

与合并版 PRD **§3.1** 一致，但 **报告生成器在 v1 仅走非 AI 路径**：模板 + `daily_analysis` → Markdown；写入 `daily_reports`（`report_type = 'fact_only'`）。

### 3.2 指标体系

**P0**：指标组 **A–D** 定义、公式、降级 与 **`二期_分析指标字典.md`** 一致。  
**P1**：指标组 **E、F** 与合并版 §6.2 一致，**不阻塞** v1 主线验收。

### 3.3 数据模型与报告章节

- `daily_analysis` / `daily_reports` 表结构 与合并版 **§3.3** 一致（`daily_reports.report_type` 在 v1 主要为 `fact_only`；`ai_enhanced` 字段预留给 AI 增强版）。  
- 报告 **固定章节** 与合并版 **§3.4** 一致，但 **不包含** 「> [AI 总结/解读]」段落；页脚 **AI 模型** 在 v1 固定为 **未启用** 或等价文案。

### 3.4 IPC（v1 必达）

| Command | 说明 |
| --- | --- |
| `generate_daily_analysis` | 按日聚合 |
| `get_daily_analysis` | 读取聚合 |
| `generate_daily_report` | **v1：`with_ai: false` 为正式路径**；`with_ai: true` 可保留占位实现，**不纳入 v1 验收** |
| `get_daily_report` | 读取 `fact_only` |
| `export_daily_report`（或等价导出命令） | 导出 `.md` |
| `get_pipeline_health` | 健康度 |
| `update_session_intent` | Intent 纠错 |
| `get_app_blacklist` / `set_app_blacklist` | 黑名单 |

具体命名以 **接口契约 Owner** 与代码为准。

## 四、事情 3：最小可用 UI + 校准（M4 + M5-校准）

### 4.1 页面与最低标准

与合并版 **§4.2** 一致：**复盘页、会话列表、设置页、健康度**。  
**设置页（v1）**：采集引擎开关、应用黑名单、**可选**：AI 总开关占位（默认关、**不产生外网**）；**不要求** API Key / 模型 / Base URL（划归 AI 增强版 PRD）。

### 4.2 v1 五步 → 四步必达

非开发者 **必须** 独立完成：

1. 打开应用 → 看到复盘页（已有报告则直接展示）  
2. 生成/刷新分析 → 看到完整 **事实层** Markdown  
3. 导出 Markdown  
4. 会话列表 → 点选会话 → 查看截图（含合理的大图预览）  

**日期切换**、健康度查看与合并版验收一致。

### 4.3 校准

| 功能 | 说明 |
| --- | --- |
| **修正 Intent** | 更新 `window_sessions.intent` + 用户 `intent_mapping` |
| **应用黑名单** | 配置持久化；**重新生成分析**后排除统计；采集侧过滤与实现一致 |

**建议增强（非 v1 必达）**：纠错后 **自动或强提示** 触发当日 `generate_daily_analysis` —— 可在 v1 补丁或 AI 增强迭代中统一产品决策。
