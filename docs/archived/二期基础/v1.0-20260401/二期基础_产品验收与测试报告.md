## 二、里程碑验收对照（Given / When / Then）

状态说明：**通过** = 已实现且与条目语义基本一致；**部分** = 简化实现或与 PRD 数值/语义有差距；**未实现** = 代码中无对应能力；**未测** = 需长时运行或手工矩阵，本次未做。

### M0 · 数据管道审计与补齐

| 编号 | 验收条目 | 状态 | 证据与说明 |
| --- | --- | --- | --- |
| M0-01 | 14 张表 + `schema_migrations` | **部分** | 迁移 v1+v2 含 `daily_analysis` / `daily_reports` / `intent_mapping` 等；**一期库原地升级**未做专项回归 |
| M0-02 | P0 四表持续写数（30min+） | **未测** | 短测与开发自测可接受；**30min+ / 8h 长稳**未纳入 |
| M0-03 | `input_metrics` / `clipboard_flows` | **部分** | 周期写入；Windows 键鼠路径与 macOS 有差异；黑名单前台剪贴板不落库（设计） |
| M0-04 | Notification Tracker | **部分** | **Windows**：可挂系统通知监听；**macOS**：`lib.rs` 未启动 `spawn_system_notification_listener`，以**短切换启发式**为主，**非** PRD 所述「通知弹出即写表」的完整语义 |
| M0-05 | Ambient Context Engine | **部分** | 定时采样；部分字段依平台可为空，与「非全 NULL」有差距时需看实机 |
| M0-06 | `get_pipeline_health` | **通过** | 六引擎状态 + 最近数据时间等 JSON |
| M0-07 | 辅助功能缺失降级 | **部分** | 主路径不崩溃；**多权限组合矩阵**未全覆盖 |
| M0-08 | 通知权限降级 | **部分** | 与健康度语义联动；macOS 真通知监听未启用时与「权限降级」条目不完全同构 |

**M0 小结**：库表 + 主采集链路 + 健康度 + 黑名单过滤已连通；**通知与环境的 PRD 级完整性**仍标为部分。

## 五、缺陷与风险（按优先级 · 二期基础）

1. **纠错闭环**：纠错后建议 **自动或强提示** 重算当日 `daily_analysis`。  
2. **M0**：通知/环境/平台差异、长稳与一期库升级专项。  
3. **黑名单**：仅 **`app_name` 精确匹配**；名称需与采样一致。  
4. **AI 能力**：整体划归 **`二期AI增强_*`**，不在基础版缺陷列表中跟踪。
